# 问题详情页闪烁问题最终解决方案

## 问题描述
进入问题详情页时，标签数量会从0闪烁变化到实际值（例如：补充(0) → 补充(1)），影响用户体验。

## 根本原因
1. 问题详情、补充列表、回答列表是分别请求的
2. 数据按顺序加载，导致UI有中间状态
3. 标签数量在数据未加载完成时就显示了默认值0

## 最终解决方案

### 方案1: 批量加载 + 延迟显示数量（已实施）

#### 核心思路
- 使用 `Promise.all` 并行加载所有数据
- 只在初始数据（featured排序）完全加载后才显示数量
- 数据未加载完成前，标签只显示名称，不显示数量

#### 关键代码修改

**1. 优化 answerTabs 的 useMemo**
```javascript
const answerTabs = React.useMemo(() => {
  // 检查初始数据是否已加载（featured排序的数据）
  const initialDataLoaded = supplementsCache.featured.loaded && answersCache.featured.loaded;
  
  // 如果初始数据未加载，不显示任何数量
  if (!initialDataLoaded) {
    return [
      t('screens.questionDetail.tabs.supplements'),
      t('screens.questionDetail.tabs.answers'),
      t('screens.questionDetail.tabs.comments'),
      t('screens.questionDetail.tabs.invite')
    ];
  }
  
  // 初始数据已加载，显示实际数量
  return [
    `${t('screens.questionDetail.tabs.supplements')} (${supplementsList.length})`,
    `${t('screens.questionDetail.tabs.answers')} (${answersTotal})`,
    `${t('screens.questionDetail.tabs.comments')} (4)`,
    t('screens.questionDetail.tabs.invite')
  ];
}, [t, supplementsList.length, answersTotal, 
    supplementsCache.featured.loaded, answersCache.featured.loaded]);
```

**2. 优化批量加载逻辑**
```javascript
// 步骤1: 并行加载所有数据
const [detailResponse, supplementsResponse, answersResponse] = await Promise.all([
  questionApi.getQuestionDetail(questionId),
  loadQuestionSupplements(questionId, 'featured', 1, true),
  answerApi.getAnswers(questionId, { sortBy: 'featured', pageNum: 1, pageSize: 10 })
]);

// 步骤2: 准备所有数据
const supplements = supplementsResponse?.data || [];
const answers = answersResponse?.data?.rows || [];
const answersTotal = answersResponse?.data?.total || 0;

// 步骤3: 批量更新所有状态（React 18+ 自动批处理）
setQuestionData(detailResponse.data);
setSupplementsCache(prevCache => ({
  ...prevCache,
  featured: { list: supplements, total: supplements.length, loaded: true, ... }
}));
setAnswersCache(prevCache => ({
  ...prevCache,
  featured: { list: answers, total: answersTotal, loaded: true, ... }
}));

// 步骤4: 启动淡入动画
setTimeout(() => {
  Animated.timing(fadeAnim, { toValue: 1, duration: 300 }).start();
}, 0);
```

## 效果对比

### 优化前
1. 显示骨架屏
2. 加载问题详情 → 显示标签"补充(0)"、"回答(0)"
3. 加载补充列表 → 标签变为"补充(1)" ⚡闪烁
4. 加载回答列表 → 标签变为"回答(5)" ⚡闪烁

### 优化后
1. 显示骨架屏
2. 并行加载所有数据（问题详情 + 补充 + 回答）
3. 所有数据准备完成 → 显示标签"补充(1)"、"回答(5)" ✅无闪烁
4. 启动淡入动画，平滑过渡

## 技术要点

### 1. 批量加载
- 使用 `Promise.all` 并行请求，减少总加载时间
- 等待所有数据返回后再更新UI

### 2. 条件渲染
- 检查 `supplementsCache.featured.loaded && answersCache.featured.loaded`
- 只在初始数据加载完成后才显示数量

### 3. React批处理
- React 18+ 自动批处理多个状态更新
- 使用 `setTimeout(..., 0)` 确保状态更新完成后再启动动画

### 4. 智能缓存
- 分排序方式缓存（featured/newest）
- 切换排序时优先使用缓存，避免重复请求

## 总结

通过批量加载 + 延迟显示数量的方案，彻底消除了标签数量闪烁问题。核心思路是：
- **等待所有初始数据加载完成后，再一次性显示完整的UI**
- 避免中间状态，确保用户看到的第一帧就是完整的数据

这个方案改动最小，不影响现有功能，且效果显著。
