# 补充标签选中指示器修复总结

## 问题描述

在问题详情页面，默认选中的是"补充"标签页，但是标签按钮下方没有显示选中的红色指示器线条。

## 问题原因

1. `activeTab` 的初始值被硬编码为 `'补充 (4)'`
2. 实际的标签文本是动态生成的，格式为 `'补充 (${supplementsList.length})'`
3. 由于补充列表的实际数量可能不是 4，导致字符串不匹配
4. 条件判断 `activeTab === tab` 返回 false，选中指示器不显示

## 修复方案

### 1. 修改初始状态

```javascript
// 修改前
const [activeTab, setActiveTab] = useState('补充 (4)');

// 修改后
const [activeTab, setActiveTab] = useState(null); // 初始为null，等待动态设置
```

### 2. 添加自动设置逻辑

```javascript
// 设置默认选中的标签页
useEffect(() => {
  if (!activeTab && answerTabs.length > 0) {
    setActiveTab(answerTabs[0]); // 默认选中第一个标签（补充）
  }
}, [answerTabs, activeTab]);
```

## 修复效果

- ✅ 页面加载时，"补充"标签自动显示红色选中指示器
- ✅ 指示器会根据实际的补充数量动态匹配
- ✅ 切换标签时，指示器正常跟随显示
- ✅ 不影响其他标签页的功能

## 技术细节

### 标签文本生成逻辑

```javascript
const answerTabs = React.useMemo(() => [
  `${t('screens.questionDetail.tabs.supplements')} (${supplementsList.length})`, 
  `${t('screens.questionDetail.tabs.answers')} (${answersTotal})`, 
  `${t('screens.questionDetail.tabs.comments')} (4)`, 
  t('screens.questionDetail.tabs.invite')
], [t, supplementsList.length, answersTotal]);
```

### 选中指示器渲染逻辑

```javascript
{answerTabs.map(tab => (
  <TouchableOpacity key={tab} style={styles.answerTabItem} onPress={() => { setActiveTab(tab); setSortFilter('精选'); }}>
    <Text style={[styles.answerTabText, activeTab === tab && styles.answerTabTextActive]}>{tab}</Text>
    {activeTab === tab && <View style={styles.answerTabIndicator} />}
  </TouchableOpacity>
))}
```

### 指示器样式

```javascript
answerTabIndicator: { 
  position: 'absolute', 
  bottom: 0, 
  left: 0, 
  right: 0, 
  height: 2, 
  backgroundColor: '#ef4444', 
  borderRadius: 1 
}
```
