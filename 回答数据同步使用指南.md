# 回答数据同步使用指南

## 功能说明

当用户在回答详情页对回答进行操作（收藏/点赞/踩）后，返回到问题详情页时，该回答在列表中的状态会自动同步更新，无需重新加载数据。

## 已实现的功能

✅ **收藏/取消收藏同步**
- 在回答详情页收藏回答后，返回问题详情页，该回答显示为已收藏状态
- 收藏数会实时更新
- 支持所有排序方式（精选/最新）的缓存同步

## 使用方法

### 对于开发者

#### 1. 添加新的操作同步

如果需要为其他操作（如点赞、点踩）添加同步功能，只需在操作成功后调用 `syncDataToQuestionDetail()`：

```javascript
// 在回答详情页 (AnswerDetailScreen.js)
const handleAnswerLike = async () => {
  try {
    // ... 点赞逻辑
    
    if (responseCode === 200) {
      toast.success('点赞成功');
      
      // 更新本地状态
      setAnswerLiked(true);
      setAnswerData(prev => ({
        ...prev,
        isLiked: true,
        likeCount: prev.likeCount + 1
      }));
      
      // 🔄 同步数据到问题详情页
      syncDataToQuestionDetail();
    }
  } catch (error) {
    // 错误处理
  }
};
```

#### 2. 扩展到其他页面

如果其他页面（如个人主页、收藏列表）也需要同步回答数据，可以：

**方案 A：使用相同的 params 传递机制**
```javascript
// 在其他页面添加 focus listener
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    const updatedAnswer = route?.params?.updatedAnswer;
    if (updatedAnswer) {
      syncAnswerData(updatedAnswer);
      navigation.setParams({ updatedAnswer: undefined });
    }
  });
  return unsubscribe;
}, [navigation, route?.params?.updatedAnswer]);
```

**方案 B：使用全局状态管理**
如果需要在多个页面之间同步，建议使用 Context API 或 Redux。

## 技术细节

### 数据流

```
用户操作 → API 调用 → 更新本地状态 → syncDataToQuestionDetail()
    ↓
navigation.navigate('QuestionDetail', { updatedAnswer })
    ↓
问题详情页 focus listener 触发
    ↓
syncAnswerData(updatedAnswer)
    ↓
更新缓存 + 更新本地状态
    ↓
React 自动重新渲染 UI
```

### 同步的数据

- **用户状态**：isLiked, isBookmarked, isDisliked
- **统计数据**：likeCount, bookmarkCount, dislikeCount
- **兼容字段**：支持多种命名格式（驼峰、下划线等）

### 性能优化

1. **无需重新请求 API**：直接更新缓存，避免网络请求
2. **批量更新**：同时更新所有排序方式的缓存
3. **防止重复处理**：处理完 updatedAnswer 后立即清除
4. **乐观更新**：操作时立即更新 UI，提升用户体验

## 常见问题

### Q1: 为什么返回后状态没有更新？

**可能原因**：
1. 操作失败，没有调用 `syncDataToQuestionDetail()`
2. navigation 对象不可用
3. 找不到问题详情页路由

**解决方法**：
- 检查控制台日志，查看是否有 "🔄 准备同步数据到问题详情页" 的日志
- 确认操作成功（看到成功提示）
- 检查 navigation.getState() 是否返回正确的路由栈

### Q2: 切换排序后状态不一致？

**可能原因**：
- `syncAnswerData` 函数没有更新所有排序方式的缓存

**解决方法**：
- 检查 `syncAnswerData` 中的 `Object.keys(newCache).forEach` 是否正确执行
- 确认所有排序方式的缓存都被更新

### Q3: 数据同步有延迟？

**说明**：
- 这是正常的，因为使用了 focus listener
- 只有当页面获得焦点时才会触发同步
- 通常延迟不超过 100ms，用户感知不到

### Q4: 如何调试同步问题？

**步骤**：
1. 打开 Metro Bundler 控制台
2. 在回答详情页操作时，查看日志：
   - `🔄 准备同步数据到问题详情页`
   - `✅ 数据已同步到问题详情页`
3. 返回问题详情页时，查看日志：
   - `🔄 检测到回答数据更新，同步到列表`
   - `✅ 在 featured 缓存中找到回答，更新数据`
   - `✅ 回答数据同步完成`
