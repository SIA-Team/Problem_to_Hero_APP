# 测试用户关注列表功能

## 快速测试步骤

### 前提条件

1. 开发服务器正在运行
2. 应用已在设备或模拟器上打开
3. 已登录账号

### 测试步骤

#### 步骤 1：查看其他用户的公开主页

1. 在应用中找到任意问题或回答
2. 点击问题或回答的作者头像/用户名
3. 进入该用户的公开主页（PublicProfileScreen）

#### 步骤 2：点击关注数

1. 在公开主页顶部，找到统计数据区域
2. 找到"关注"项（显示为数字，例如 "0"）
3. 点击"关注"统计数字

#### 步骤 3：查看关注列表

1. 应该跳转到"XXX 的关注"页面
2. 页面标题显示该用户的名字，例如"UserMkhzBHz 的关注"
3. 如果该用户有关注的人，会显示关注列表
4. 如果该用户没有关注任何人，会显示"还没有关注任何人"的提示

#### 步骤 4：测试交互功能

1. **下拉刷新**：在列表顶部下拉，释放后刷新数据
2. **滚动加载**：滚动到列表底部，自动加载更多用户
3. **点击用户**：点击任意用户项，跳转到该用户的公开资料页面
4. **返回**：点击左上角返回按钮，返回上一页

## 可能遇到的问题

### 问题 1：点击关注数没有反应

**原因**：可能是路由没有正确注册

**解决方案**：
1. 确认 App.js 中已导入 UserFollowingScreen
2. 确认已添加 `<Stack.Screen name="UserFollowing" component={UserFollowingScreen} />`
3. 重启开发服务器

### 问题 2：页面显示空白或报错

**原因**：可能是接口返回的数据格式不符合预期

**解决方案**：
1. 查看控制台日志，检查接口返回的数据格式
2. UserFollowingScreen 已经实现了多种数据格式的兼容处理
3. 如果数据格式特殊，可能需要调整 `extractFollowingFromPayload` 函数

### 问题 3：翻译文本显示为 key

**原因**：翻译文件可能没有正确更新

**解决方案**：
1. 确认 `src/i18n/locales/zh.json` 中已添加 `userFollowing` 部分
2. 确认 `src/i18n/locales/en.json` 中已添加 `userFollowing` 部分
3. 重启应用

### 问题 4：接口调用失败

**原因**：接口路径可能不正确

**解决方案**：
1. 查看控制台的接口调用日志
2. 确认接口路径：`/qa-hero-app-user/app/user/follow/:userId/following`
3. 确认 userId 参数正确传递
4. 如果后端接口路径不同，需要在 `src/config/api.js` 中修改

### 问题 5：页面标题显示"用户 的关注"

**原因**：username 参数没有正确传递

**解决方案**：
1. 检查 PublicProfileScreen 中跳转时是否传递了 username 参数
2. 确认 userData?.username 有值
3. 如果没有 username，会显示默认的"关注列表"标题
