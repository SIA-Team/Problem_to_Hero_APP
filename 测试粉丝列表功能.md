# 测试粉丝列表功能

## 快速测试步骤

### 1. 重启开发服务器（如果需要）

由于修改了 App.js 和添加了新的 Screen，建议重启开发服务器：

```bash
# 如果服务器正在运行，先停止（Ctrl+C）
# 然后重新启动
npm start
```

或者在 Expo 开发工具中按 `r` 重新加载应用。

### 2. 在应用中测试

#### 步骤 A：导航到个人资料页面
1. 打开应用
2. 点击底部导航栏的"我的"标签
3. 进入个人资料页面

#### 步骤 B：点击粉丝数
1. 在个人资料页面顶部，找到统计数据区域
2. 找到"粉丝"项（显示为 "1.2k"）
3. 点击"粉丝"统计数字

#### 步骤 C：查看粉丝列表
1. 应该跳转到"粉丝列表"页面
2. 页面标题显示"粉丝列表"
3. 如果有粉丝数据，会显示粉丝列表
4. 如果没有粉丝，会显示"还没有粉丝"的提示

#### 步骤 D：测试交互功能
1. **下拉刷新**：在列表顶部下拉，释放后刷新数据
2. **滚动加载**：滚动到列表底部，自动加载更多粉丝
3. **点击粉丝**：点击任意粉丝项，跳转到该用户的公开资料页面
4. **返回**：点击左上角返回按钮，返回个人资料页面

## 可能遇到的问题

### 问题 1：点击粉丝数没有反应

**原因**：可能是路由没有正确注册

**解决方案**：
1. 确认 App.js 中已导入 FansScreen
2. 确认已添加 `<Stack.Screen name="Fans" component={FansScreen} />`
3. 重启开发服务器

### 问题 2：页面显示空白或报错

**原因**：可能是接口返回的数据格式不符合预期

**解决方案**：
1. 查看控制台日志，检查接口返回的数据格式
2. FansScreen 已经实现了多种数据格式的兼容处理
3. 如果数据格式特殊，可能需要调整 `extractFansFromPayload` 函数

### 问题 3：翻译文本显示为 key

**原因**：翻译文件可能没有正确更新

**解决方案**：
1. 确认 `src/i18n/locales/zh.json` 中已添加 `fans` 部分
2. 确认 `src/i18n/locales/en.json` 中已添加 `fans` 部分
3. 重启应用

### 问题 4：接口调用失败

**原因**：接口路径可能不正确

**解决方案**：
1. 查看控制台的接口调用日志
2. 确认接口路径：`/qa-hero-app-user/app/user/follow/my/followers`
3. 如果后端接口路径不同，需要在 `src/config/api.js` 中修改
