# 标签数量闪烁问题修复

## 问题描述

进入问题详情页时，标签文本会闪烁：
- 初始显示："补充 (0)"
- 数据加载后："补充 (1)"
- 这个变化导致视觉上的闪烁

## 问题原因

1. 标签文本包含动态数量
2. 数据加载是异步的
3. 数量从 0 变为实际值时，标签文本改变
4. React 重新渲染导致视觉闪烁

## 解决方案

### 方案 1：延迟显示数量（已实施）✅

**核心思路**：在数据加载完成前不显示数量

```javascript
const answerTabs = React.useMemo(() => {
  const supplementsLoaded = supplementsCache.featured.loaded || supplementsCache.newest.loaded;
  const answersLoaded = answersCache.featured.loaded || answersCache.newest.loaded;
  
  return [
    supplementsLoaded 
      ? `${t('screens.questionDetail.tabs.supplements')} (${supplementsList.length})` 
      : t('screens.questionDetail.tabs.supplements'),  // 不显示数量
    answersLoaded 
      ? `${t('screens.questionDetail.tabs.answers')} (${answersTotal})` 
      : t('screens.questionDetail.tabs.answers'),  // 不显示数量
    `${t('screens.questionDetail.tabs.comments')} (4)`, 
    t('screens.questionDetail.tabs.invite')
  ];
}, [t, supplementsList.length, answersTotal, supplementsCache, answersCache]);
```

**效果**：
- ✅ 初始显示："补充"（无数量）
- ✅ 数据加载后："补充 (1)"
- ✅ 只有一次文本变化，且是从无到有
- ✅ 视觉上更自然，不会感觉到闪烁

### 方案 2：使用占位符（备选）

**核心思路**：始终显示数量位置，用占位符表示加载中

```javascript
supplementsLoaded 
  ? `${t('screens.questionDetail.tabs.supplements')} (${supplementsList.length})` 
  : `${t('screens.questionDetail.tabs.supplements')} (...)`  // 使用 ... 占位
```

**效果**：
- 初始显示："补充 (...)"
- 数据加载后："补充 (1)"
- 文本长度变化较小

### 方案 3：预加载数据（长期优化）

**核心思路**：在列表页就预加载问题详情

```javascript
// 在 HomeScreen 或列表页
const prefetchQuestionDetail = async (questionId) => {
  const data = await questionApi.getQuestionDetail(questionId);
  // 缓存数据
  cache.set(`question_${questionId}`, data);
};
```

## 优化细节

### 1. 平滑更新标签

使用 `requestAnimationFrame` 确保在下一帧更新：

```javascript
if (currentTabIndex !== -1 && answerTabs[currentTabIndex] !== activeTab) {
  requestAnimationFrame(() => {
    setActiveTab(answerTabs[currentTabIndex]);
  });
}
```

### 2. 智能判断加载状态

检查缓存的 `loaded` 标志：

```javascript
const supplementsLoaded = supplementsCache.featured.loaded || supplementsCache.newest.loaded;
```

### 3. 保持选中状态

通过提取标签类型（去掉数量）来匹配：

```javascript
const activeType = activeTab.split(' (')[0].trim();
const tabType = tab.split(' (')[0].trim();
return activeType === tabType;
```

## 技术要点

### 1. useMemo 优化
- 避免不必要的重新计算
- 依赖项包含缓存状态

### 2. 条件渲染
- 根据加载状态决定是否显示数量
- 保持标签文本的稳定性

### 3. requestAnimationFrame
- 确保更新在下一帧执行
- 避免同步更新导致的闪烁
