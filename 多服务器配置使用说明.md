# 多服务器配置使用说明

## 概述

项目已配置好多服务器架构，当前所有接口都使用生产服务器 `http://8.146.230.62:8080`，开发服务器 `http://123.144.149.59:30560` 已配置为备用，可以随时切换特定接口。

## 当前状态

### 服务器地址
- **Server2 (生产服务器 - 当前使用)**: `http://8.146.230.62:8080` - **所有接口当前都使用这个**
- **Server1 (开发服务器 - 备用)**: `http://123.144.149.59:30560` - **已配置好，可随时切换**
- **Mock服务器**: `https://m1.apifoxmock.com/m1/7857964-7606903-default`

### 当前接口分配

**所有接口目前都使用生产服务器** (`http://8.146.230.62:8080`)，包括：
- 认证相关接口（登录、注册、Token等）
- 用户相关接口（用户资料、头像等）
- 分类相关接口
- 问题相关接口（问题列表、详情、发布等）
- 回答相关接口（回答列表、点赞、收藏、采纳等）
- 补充相关接口

## 如何切换接口到开发服务器

当你需要将某些接口切换到开发服务器 `http://123.144.149.59:30560` 时，只需要在 `src/config/env.js` 文件中取消对应接口的注释并设置为 `'server1'`：

### 示例：切换登录接口到开发服务器

```javascript
const API_SERVER_CONFIG = {
  // 取消注释并设置为 server1
  '/app/user/auth/login': 'server1',         // 登录接口使用开发服务器
  
  // 其他接口保持注释状态，继续使用生产服务器
};
```

### 示例：切换所有认证接口到开发服务器

```javascript
const API_SERVER_CONFIG = {
  // 认证相关接口 - 切换到开发服务器
  '/app/user/auth/register': 'server1',      // 注册接口
  '/app/user/auth/login': 'server1',         // 登录接口
  '/app/user/auth/token-login': 'server1',   // Token登录
  '/app/user/auth/logout': 'server1',        // 退出登录
  '/app/user/auth/password': 'server1',      // 修改密码
};
```

### 示例：切换问题相关接口到开发服务器

```javascript
const API_SERVER_CONFIG = {
  // 问题相关接口 - 切换到开发服务器
  '/app/content/question/list': 'server1',           // 问题列表
  '/app/content/question/detail': 'server1',         // 问题详情
  '/app/content/question/publish': 'server1',        // 发布问题
  '/app/content/question/draft': 'server1',          // 草稿相关
};
```

## 配置选项说明

- **不配置** (注释状态): 使用生产服务器 (`http://8.146.230.62:8080`) - 默认
- **`'server1'`**: 使用开发服务器 (`http://123.144.149.59:30560`)
- **`'server2'`**: 明确指定使用生产服务器（通常不需要，因为这是默认值）
- **`'mock'`**: 使用Mock服务器

## 动态路径支持

对于包含动态参数的接口路径，可以使用通配符 `*`：

```javascript
// 匹配 /app/content/answer/question/123/accept/456
'/app/content/answer/question/*/accept/*': 'server1',
```

- 统一使用: 生产服务器 (`http://8.146.230.62:8080`)

## 快速切换指南

### 1. 切换单个接口到开发服务器
找到对应接口，取消注释并设置为 `'server1'`：
```javascript
'/app/user/auth/login': 'server1',
```

### 2. 切换一组接口到开发服务器
取消注释相关的一组接口：
```javascript
// 用户相关接口 - 切换到开发服务器
'/app/user/profile': 'server1',
'/app/user/profile/me': 'server1',
'/app/user/profile/username': 'server1',
'/app/user/profile/avatar': 'server1',
```

### 3. 临时使用Mock
```javascript
'/app/content/question/list': 'mock',
```

### 4. 切换回生产服务器
重新注释掉对应行即可：
```javascript
// '/app/user/auth/login': 'server1',  // 注释掉，回到默认的生产服务器
```

## 注意事项

1. **当前状态**: 所有接口都使用生产服务器，开发服务器配置已准备好
2. **生产环境**: 生产环境下统一使用生产服务器地址
3. **配置验证**: 切换后建议测试相关接口确保正常工作
4. **注释管理**: 使用注释来控制是否启用特定服务器配置

## 常见使用场景

### 场景1：测试新开发的接口
如果某个接口在开发服务器上刚开发完成，需要测试：
```javascript
'/app/content/new/api': 'server1',  // 使用开发服务器测试新接口
```

### 场景2：生产服务器某接口有问题，临时切换到开发服务器
```javascript
'/app/content/problem/api': 'server1',  // 临时使用开发服务器
```

### 场景3：前端独立开发，使用Mock数据
```javascript
'/app/content/question/list': 'mock',  // 使用Mock数据进行前端开发
```
