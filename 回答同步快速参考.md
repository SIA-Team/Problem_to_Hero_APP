# 回答数据同步 - 快速参考

## 🎯 核心概念

用户在回答详情页的操作会自动同步到问题详情页，无需重新加载数据。

## 📝 实现步骤

### 1️⃣ 回答详情页 - 发送数据

```javascript
// 在操作成功后调用
if (responseCode === 200) {
  toast.success('操作成功');
  
  // 更新本地状态...
  
  // 🔄 同步数据
  syncDataToQuestionDetail();
}
```

### 2️⃣ 问题详情页 - 接收数据

```javascript
// focus listener 自动处理
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    const updatedAnswer = route?.params?.updatedAnswer;
    if (updatedAnswer) {
      syncAnswerData(updatedAnswer);
      navigation.setParams({ updatedAnswer: undefined });
    }
  });
  return unsubscribe;
}, [navigation, route?.params?.updatedAnswer]);
```

## 🔧 关键函数

### syncDataToQuestionDetail()
**位置**：`src/screens/AnswerDetailScreen.js`  
**作用**：将更新的回答数据传递给问题详情页  
**调用时机**：任何操作成功后（收藏、点赞、点踩等）

### syncAnswerData(updatedAnswer)
**位置**：`src/screens/QuestionDetailScreen.js`  
**作用**：更新缓存中的回答数据  
**调用时机**：页面获得焦点且检测到 updatedAnswer 参数

## 📊 同步的数据字段

| 字段类型 | 字段名 | 说明 |
|---------|--------|------|
| 用户状态 | isLiked, liked | 是否已点赞 |
| 用户状态 | isBookmarked, bookmarked, collected | 是否已收藏 |
| 用户状态 | isDisliked, disliked | 是否已点踩 |
| 统计数据 | likeCount, like_count, likes | 点赞数 |
| 统计数据 | bookmarkCount, bookmark_count, bookmarks | 收藏数 |
| 统计数据 | dislikeCount, dislike_count, dislikes | 点踩数 |

## ✅ 测试检查清单

- [ ] 收藏后返回，状态正确
- [ ] 取消收藏后返回，状态正确
- [ ] 收藏数正确更新
- [ ] 切换排序方式，状态一致
- [ ] 多次操作，所有回答状态正确
- [ ] 网络错误时，状态不变

## 🐛 调试技巧

### 查看日志
```javascript
// 回答详情页
🔄 准备同步数据到问题详情页
✅ 数据已同步到问题详情页

// 问题详情页
🔄 检测到回答数据更新，同步到列表
✅ 在 featured 缓存中找到回答，更新数据
✅ 回答数据同步完成
```

### 常见问题
1. **状态没更新** → 检查是否调用了 `syncDataToQuestionDetail()`
2. **排序不一致** → 检查是否更新了所有缓存
3. **数据错误** → 检查 updatedAnswer 对象的字段
