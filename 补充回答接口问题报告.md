# 补充回答发布接口问题报告

## 问题描述

在调用补充回答发布接口时，服务器返回 500 错误，提示 `question_id` 字段不能为 null。

## 接口文档

根据后端提供的接口文档：

- **接口地址**: `POST /app/content/answer-supplement/answer/{answerId}`
- **路径参数**: `answerId` (int, 必传) - 回答ID
- **请求体参数**: 
  ```json
  {
    "content": "string, 必传"
  }
  ```
- **响应格式**: 
  ```json
  {
    "code": 200,
    "msg": "",
    "data": 0
  }
  ```

## 实际错误

### 请求示例
```http
POST /app/content/answer-supplement/answer/123
Content-Type: application/json

{
  "content": "这是一条补充回答"
}
```

### 错误响应
```json
{
  "code": 500,
  "msg": "Error updating database. Cause: java.sql.SQLIntegrityConstraintViolationException: Column 'question_id' cannot be null"
}
```

### 错误堆栈
```
java.sql.SQLIntegrityConstraintViolationException: Column 'question_id' cannot be null
The error may exist in com/sia/content/mapper/AnswerSupplementMapper.java
The error occurred while setting parameters
SQL: insert into answer_supplement(answer_id,question_id,user_id,content,like_count,dislike_count,comment_count,audit_status,del_flag,create_time,update_time) values(?,?,?,?,?,?,?,?,?,?,?)
```

## 问题分析

1. **接口文档不完整**: 文档中没有要求传递 `question_id` 参数
2. **数据库约束**: `answer_supplement` 表的 `question_id` 字段设置为 `NOT NULL`
3. **后端逻辑缺失**: 后端在插入数据时没有自动填充 `question_id`
4. **文档与实现不一致**: 接口文档和实际实现存在差异

## 解决方案

### 方案1：后端修复（推荐）

**优点**:
- 保持接口文档的准确性
- 前端无需修改
- 符合 RESTful 设计原则（通过 answerId 可以查询到 questionId）

**实现建议**:
```java
// 在 AnswerSupplementService 或 AnswerSupplementMapper 中
public int insertAnswerSupplement(AnswerSupplement answerSupplement) {
    // 1. 根据 answerId 查询对应的 answer
    Answer answer = answerMapper.selectById(answerSupplement.getAnswerId());
    
    // 2. 自动填充 questionId
    if (answer != null) {
        answerSupplement.setQuestionId(answer.getQuestionId());
    }
    
    // 3. 插入补充回答
    return answerSupplementMapper.insert(answerSupplement);
}
```

### 方案2：更新接口文档并要求前端传递 questionId

**优点**:
- 实现简单，无需修改后端逻辑

**缺点**:
- 需要更新接口文档
- 前端需要确保能获取到 questionId
- 增加了前端的复杂度
- 不符合 RESTful 设计原则

**接口文档更新**:
```json
{
  "content": "string, 必传",
  "questionId": "integer, 必传"
}
```

**前端修改**:
```javascript
const response = await answerApi.publishSupplementAnswer(answerId, {
  content: supplementAnswerText.trim(),
  questionId: questionId  // 需要从上下文获取
});
```

## 建议

**强烈建议采用方案1**，理由如下：

1. **数据一致性**: `answerId` 和 `questionId` 存在固定的关联关系，后端应该自动维护
2. **接口简洁性**: 前端只需要知道要补充哪个回答，不需要关心问题ID
3. **减少错误**: 避免前端传递错误的 questionId
4. **符合设计原则**: RESTful API 应该通过资源关系自动推导相关信息
