## ⚠️ 常见问题

### Q1: 构建失败怎么办？

**检查**：
1. EAS CLI 版本是否最新
2. 网络连接是否正常
3. 查看错误日志

**解决**：
```bash
# 更新 EAS CLI
npm install -g eas-cli

# 重新登录
eas login

# 重新构建
eas build --platform android --profile production
```

### Q2: Android 构建成功但无法安装？

**可能原因**：
1. 设备版本太低（< Android 6.0）
2. 签名问题
3. 存储空间不足

**解决**：
- 检查设备 Android 版本
- 确认 APK 签名正确
- 清理设备存储空间

### Q3: iOS 构建需要什么？

**必需**：
1. Apple Developer 账号（$99/年）
2. Bundle Identifier 已注册
3. 证书和描述文件（EAS 自动处理）

**可选**：
- Mac 电脑（使用 EAS Build 不需要）
- Xcode（使用 EAS Build 不需要）

### Q4: 用户报告无法自动注册？

**排查步骤**：
1. 确认用户设备版本
2. 确认网络连接
3. 让用户尝试"使用设备登录"按钮
4. 使用网络诊断工具

**常见原因**：
- 网络无法访问服务器
- 防火墙阻止
- 设备时间不正确

## 📊 版本管理

### 当前版本

```json
{
  "version": "1.0.0",
  "android": {
    "versionCode": 3  // 自动递增
  },
  "ios": {
    "buildNumber": "1"
  }
}
```

### 版本号规则

- **version**: 显示给用户的版本号（如 1.0.0）
- **versionCode** (Android): 内部版本号，每次构建自动递增
- **buildNumber** (iOS): 构建号，每次构建需要递增

### 更新版本

```json
// 小更新（bug 修复）
"version": "1.0.1"

// 功能更新
"version": "1.1.0"

// 大版本更新
"version": "2.0.0"
```
