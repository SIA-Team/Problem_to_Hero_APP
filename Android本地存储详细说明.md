# Android 应用本地存储详细说明

## 📱 应用使用的存储方式

您的应用使用 **AsyncStorage**（React Native 的本地存储方案）来保存数据。

### 存储的数据类型

#### 1. 用户认证信息
- `authToken` - 登录令牌
- `refreshToken` - 刷新令牌
- `userInfo` - 用户基本信息（JSON 格式）

#### 2. 设备信息
- `deviceFingerprint` - 设备指纹（用于自动登录）

#### 3. SuperLike 积分系统
- `@superlike_balance` - 超级赞余额
- `@superlike_transactions` - 交易历史记录
- `@superlike_migrated` - 数据迁移标志

#### 4. 其他可能的缓存数据
- 用户名修改时间
- 应用配置信息
- 临时缓存数据

---

## 🗑️ 卸载应用后的数据清理

### ✅ 会被自动清除的数据

在 **Android 系统**中，当您卸载应用时，以下数据会被**自动完全清除**：

1. **AsyncStorage 存储的所有数据**
   - 所有 token、用户信息
   - 设备指纹
   - SuperLike 积分余额和交易历史
   - 所有应用内缓存

2. **应用的内部存储**
   - 应用安装目录下的所有文件
   - 应用缓存目录的所有文件

3. **应用的私有数据**
   - SharedPreferences（AsyncStorage 在 Android 上的底层实现）
   - 数据库文件
   - 应用专属的文件

### ❌ 不会被清除的数据

以下数据**不会**随应用卸载而清除：

1. **外部存储的文件**（如果应用保存了文件到公共目录）
   - 下载的图片、文档等（保存在 Downloads、Pictures 等公共目录）
   - 注：您的应用目前**没有**使用外部存储

2. **服务器端数据**
   - 用户账号信息
   - 问题、回答、评论等内容
   - 用户关系（关注、粉丝）
   - 这些数据保存在服务器上，卸载应用不会影响

## 🔄 重新安装后的状态

### 场景 1: 卸载后重新安装

**结果**: 应用会像首次安装一样启动

- ✅ 所有本地数据已清除
- ✅ 设备指纹会重新生成（新的指纹）
- ✅ 会触发自动注册流程，创建**新账号**
- ❌ 无法自动登录到原来的账号

### 场景 2: 想要保留原账号

如果您想在重新安装后继续使用原来的账号，需要：

1. **卸载前记住用户名和密码**
2. 重新安装后，使用用户名和密码手动登录
3. 或者联系后端开发者实现账号绑定功能（如手机号、邮箱绑定）

## 🔒 Android 数据清理机制说明

### AsyncStorage 在 Android 上的实现

AsyncStorage 在 Android 上使用 **SharedPreferences** 存储数据：

- **存储位置**: `/data/data/com.qa.app/shared_prefs/`
- **权限**: 仅应用自己可以访问
- **清理时机**: 
  - 卸载应用时自动清除
  - 在系统设置中"清除数据"时清除
  - 应用内调用清除 API 时清除

### Android 系统保证

根据 Android 官方文档：

> 当应用被卸载时，系统会自动删除应用的所有私有数据，包括：
> - 内部存储目录（/data/data/包名/）
> - 应用缓存目录
> - SharedPreferences
> - 数据库文件

**结论**: 您的应用使用的 AsyncStorage 数据会在卸载时被**完全清除**，不会留下任何痕迹。

## 📊 数据清理验证方法

如果您想验证数据是否被清除，可以：

### 方法 1: 使用 ADB 命令（需要开发者模式）

```bash
# 查看应用数据目录
adb shell ls -la /data/data/com.qa.app/

# 卸载应用
adb uninstall com.qa.app

# 再次查看（会提示目录不存在）
adb shell ls -la /data/data/com.qa.app/
```

### 方法 2: 通过应用行为验证

1. 安装应用并登录
2. 记住当前的 SuperLike 余额
3. 卸载应用
4. 重新安装应用
5. 观察：
   - ✅ 需要重新登录（说明 token 已清除）
   - ✅ SuperLike 余额为 0（说明本地数据已清除）
   - ✅ 会生成新的设备指纹（说明设备信息已清除）

## 💡 常见问题

### Q1: 卸载应用后，我的账号还存在吗？
**A**: 是的，您的账号信息保存在服务器上，卸载应用不会删除账号。但本地的登录状态会被清除。

### Q2: 重新安装后能自动登录吗？
**A**: 不能。因为本地的 token 和设备指纹已被清除，需要重新登录。

### Q3: SuperLike 积分会丢失吗？
**A**: 本地记录的积分会清除。如果积分数据也保存在服务器上，重新登录后可以恢复；如果仅保存在本地，则会丢失。

### Q4: 如何在卸载前备份数据？
**A**: 目前应用没有数据导出功能。建议记住用户名和密码，重新安装后手动登录。

### Q5: 清除应用数据和卸载应用有什么区别？
**A**: 
- **清除数据**: 在系统设置中清除应用数据，应用仍然安装，但所有本地数据被清除
- **卸载应用**: 删除应用本身和所有数据

两者对本地数据的清理效果是一样的。
