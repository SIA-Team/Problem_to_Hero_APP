# 回答详情与问题详情数据同步方案

## 问题描述

需要实现：在回答详情页操作（点赞、收藏、踩）后，问题详情页的回答列表中对应回答的数据也能同步更新，但不要跳转回问题详情页。

## 解决方案

### 核心思路

使用 React Navigation 的参数传递机制，通过 `setParams` 更新参数而不触发页面跳转。

### 实现步骤

#### 1. 回答详情页（AnswerDetailScreen.js）

**修改 `syncDataToQuestionDetail` 函数：**

```javascript
// 同步数据到问题详情页（不跳转，只更新参数）
const syncDataToQuestionDetail = () => {
  if (!answerData || !navigation) return;
  
  const routes = navigation.getState()?.routes;
  const questionDetailRoute = routes?.find(r => r.name === 'QuestionDetail');
  
  if (questionDetailRoute) {
    const updatedAnswer = {
      id: answerData.id,
      isLiked: answerLiked,
      liked: answerLiked,
      isBookmarked: answerBookmarked,
      bookmarked: answerBookmarked,
      collected: answerBookmarked,
      isDisliked: answerDisliked,
      disliked: answerDisliked,
      likeCount: answerData.likeCount || answerData.like_count || answerData.likes || 0,
      like_count: answerData.likeCount || answerData.like_count || answerData.likes || 0,
      likes: answerData.likeCount || answerData.like_count || answerData.likes || 0,
      bookmarkCount: answerData.bookmarkCount || answerData.bookmark_count || answerData.bookmarks || 0,
      bookmark_count: answerData.bookmarkCount || answerData.bookmark_count || answerData.bookmarks || 0,
      bookmarks: answerData.bookmarkCount || answerData.bookmark_count || answerData.bookmarks || 0,
      dislikeCount: answerData.dislikeCount || answerData.dislike_count || answerData.dislikes || 0,
      dislike_count: answerData.dislikeCount || answerData.dislike_count || answerData.dislikes || 0,
      dislikes: answerData.dislikeCount || answerData.dislike_count || answerData.dislikes || 0,
    };
    
    // 使用 setParams 更新参数，不触发导航跳转
    navigation.setParams({
      updatedAnswer: updatedAnswer
    });
  }
};
```

**关键改动：**
- 将 `navigation.navigate()` 改为 `navigation.setParams()`
- 只传递 `updatedAnswer`，不传递其他参数
- 这样只更新参数，不会触发页面跳转

**在收藏操作成功后调用：**

```javascript
if (responseCode === 200) {
  const successMessage = newBookmarked ? '收藏成功' : '取消收藏成功';
  toast.success(successMessage);
  
  if (responseDataField) {
    // ... 更新本地数据
  }
  
  // 同步数据到问题详情页（不跳转）
  syncDataToQuestionDetail();
}
```

#### 2. 问题详情页（QuestionDetailScreen.js）

**监听参数变化：**

问题详情页已经实现了监听机制：

```javascript
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    // 检查是否有从回答详情页返回的更新数据
    const updatedAnswer = route?.params?.updatedAnswer;
    if (updatedAnswer) {
      console.log('🔄 检测到回答数据更新，同步到列表:', updatedAnswer.id);
      syncAnswerData(updatedAnswer);
      
      // 清除 params 中的 updatedAnswer，避免重复处理
      navigation.setParams({ updatedAnswer: undefined });
    }
  });

  return unsubscribe;
}, [navigation, answersCache, route?.params?.id, route?.params?.updatedAnswer]);
```

**同步数据到列表：**

```javascript
const syncAnswerData = (updatedAnswer) => {
  if (!updatedAnswer || !updatedAnswer.id) return;
  
  // 更新缓存中的数据
  setAnswersCache(prevCache => {
    const newCache = { ...prevCache };
    
    Object.keys(newCache).forEach(sortBy => {
      const cacheData = newCache[sortBy];
      if (cacheData.list && cacheData.list.length > 0) {
        const answerIndex = cacheData.list.findIndex(a => a.id === updatedAnswer.id);
        
        if (answerIndex !== -1) {
          const updatedList = [...cacheData.list];
          updatedList[answerIndex] = {
            ...updatedList[answerIndex],
            // 用户状态
            isLiked: updatedAnswer.isLiked,
            liked: updatedAnswer.isLiked,
            isBookmarked: updatedAnswer.isBookmarked,
            bookmarked: updatedAnswer.isBookmarked,
            collected: updatedAnswer.isBookmarked,
            isDisliked: updatedAnswer.isDisliked,
            disliked: updatedAnswer.isDisliked,
            // 统计数据
            likeCount: updatedAnswer.likeCount,
            like_count: updatedAnswer.likeCount,
            likes: updatedAnswer.likeCount,
            bookmarkCount: updatedAnswer.bookmarkCount,
            bookmark_count: updatedAnswer.bookmarkCount,
            bookmarks: updatedAnswer.bookmarkCount,
            dislikeCount: updatedAnswer.dislikeCount,
            dislike_count: updatedAnswer.dislikeCount,
            dislikes: updatedAnswer.dislikeCount,
          };
          
          newCache[sortBy] = {
            ...cacheData,
            list: updatedList
          };
        }
      }
    });
    
    return newCache;
  });
  
  // 更新状态
  setAnswerLiked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isLiked || false
  }));
  
  setAnswerBookmarked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isBookmarked || false
  }));
  
  setAnswerDisliked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isDisliked || false
  }));
};
```

## 工作流程

1. 用户在回答详情页点击收藏按钮
2. 调用收藏接口，成功后更新本地状态
3. 调用 `syncDataToQuestionDetail()` 函数
4. 使用 `navigation.setParams()` 更新问题详情页的参数
5. 问题详情页的 `useEffect` 监听到参数变化
6. 调用 `syncAnswerData()` 更新列表中的数据
7. 用户返回问题详情页时，看到的是最新数据

## 优势

1. **无需额外接口调用**：复用现有的接口，不增加服务器负担
2. **实时同步**：操作后立即同步，无延迟
3. **不影响用户体验**：不会强制跳转，用户可以继续浏览回答详情
4. **数据一致性**：确保两个页面的数据完全一致
5. **适用于所有操作**：点赞、收藏、踩都可以使用同样的机制

## 扩展应用

这个方案同样适用于：
- 点赞操作
- 点踩操作
- 评论数更新
- 分享数更新
- 任何需要跨页面同步的数据

只需要在对应的操作成功后调用 `syncDataToQuestionDetail()` 即可。

## 注意事项

1. 确保 `answerData` 包含最新的统计数据
2. 使用 `setParams` 而不是 `navigate` 避免跳转
3. 在问题详情页处理完更新后，清除 `updatedAnswer` 参数避免重复处理
4. 所有字段都做了多种命名方式的兼容（如 `likeCount`、`like_count`、`likes`）
