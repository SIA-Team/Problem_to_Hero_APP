# 真机调试 API 请求指南

## 方法：使用内置 API 调试工具（最简单）✅

我已经为你创建了一个内置的 API 调试页面，可以实时查看所有 API 请求和响应。

### 使用步骤：

1. **打开应用**
   - 在真机上打开应用（生产环境版本）

2. **进入 API 调试页面**
   - 点击底部导航栏的"我的"
   - 点击右上角设置图标
   - 找到"API 调试"选项（蓝色图标）
   - 点击进入

3. **查看 API 请求**
   - 页面顶部显示当前使用的服务器地址
   - 下方实时显示所有 API 请求日志
   - 绿色边框 = 成功请求
   - 红色边框 = 失败请求
   - 橙色边框 = 请求中

4. **查看详细信息**
   - 点击任意请求可展开详情
   - 可以看到：
     - 完整 URL
     - 请求参数
     - 请求体
     - 响应数据
     - 错误信息（如果有）
     - 请求耗时

5. **分享日志**
   - 展开请求详情后
   - 点击底部"分享日志"按钮
   - 可以通过微信、邮件等方式分享给开发者

6. **清除日志**
   - 点击右上角垃圾桶图标清除所有日志

7. **暂停/继续记录**
   - 点击右上角暂停/播放按钮
   - 暂停后不会记录新的请求

### 功能特点：

✅ **实时监控**：自动记录所有 API 请求
✅ **详细信息**：显示完整的请求和响应数据
✅ **错误追踪**：清晰显示错误信息和状态码
✅ **服务器地址**：显示当前使用的服务器地址
✅ **一键分享**：可以将日志分享给开发者
✅ **无需连接电脑**：直接在手机上查看

### 查看内容：

1. **当前服务器地址**
   ```
   当前服务器: http://8.146.230.62:8080
   内容服务器: http://8.146.230.62:8080
   ```

2. **请求信息**
   - HTTP 方法（GET、POST 等）
   - 完整 URL
   - 请求时间
   - 请求参数
   - 请求体数据

3. **响应信息**
   - HTTP 状态码
   - 响应数据
   - 请求耗时

4. **错误信息**（如果请求失败）
   - 错误消息
   - 错误代码
   - 服务器返回的错误详情

### 使用场景：

1. **检查服务器地址**
   - 确认生产环境使用的是哪个服务器
   - 验证服务器切换是否生效

2. **调试接口问题**
   - 查看接口返回的具体错误
   - 检查请求参数是否正确
   - 确认 token 是否正确传递

3. **性能分析**
   - 查看每个请求的耗时
   - 找出慢接口

4. **问题反馈**
   - 截图或分享日志给开发者
   - 提供详细的错误信息

## 其他方法（需要连接电脑）

### 方法二：使用 React Native Debugger

1. **下载安装**
   - 访问：https://github.com/jhen0409/react-native-debugger/releases
   - 下载 Windows 版本并安装

2. **连接真机**
   - 确保手机和电脑在同一 WiFi
   - 打开应用，摇晃手机
   - 选择"Debug"或"启用远程调试"

3. **查看网络请求**
   - 在 Debugger 中点击"Network"标签
   - 查看所有 API 请求

### 方法三：使用 Reactotron

1. **下载安装**
   - 访问：https://github.com/infinitered/reactotron/releases
   - 下载 Windows 版本

2. **配置项目**（需要修改代码）
   - 安装依赖：`npm install --save-dev reactotron-react-native`
   - 添加配置文件
   - 重新构建应用

## 常见问题

**Q: 为什么看不到请求日志？**
A: 确保应用已经发起了 API 请求。尝试刷新页面或执行一些操作（如登录、浏览问题等）。

**Q: 日志太多怎么办？**
A: 点击右上角垃圾桶图标清除日志，或者点击暂停按钮停止记录。

**Q: 如何查看历史请求？**
A: 日志会保留最近 50 条请求，向下滚动可以查看历史记录。

**Q: 可以导出日志吗？**
A: 点击展开请求详情，然后点击"分享日志"按钮，可以通过各种方式分享。
