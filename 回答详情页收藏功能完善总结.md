# 回答详情页收藏功能完善总结

## 功能概述

完善了回答详情页的收藏功能，确保点击收藏按钮时正确调用收藏和取消收藏的API接口，并实现了完整的状态管理和用户体验优化。

## 主要改进

### 1. 状态管理优化

**问题**：原来的状态初始化依赖于可能未完成的 `answer` 对象，导致状态不准确。

**解决方案**：
```javascript
// 延迟初始化，避免依赖未完成的answer对象
const [answerLiked, setAnswerLiked] = useState(false);
const [answerBookmarked, setAnswerBookmarked] = useState(false);
const [answerDisliked, setAnswerDisliked] = useState(false);

// 通过useEffect正确初始化用户状态
useEffect(() => {
  if (answer) {
    setAnswerLiked(answer.isLiked || answer.liked || false);
    setAnswerBookmarked(answer.isBookmarked || answer.bookmarked || answer.collected || false);
    setAnswerDisliked(answer.isDisliked || answer.disliked || false);
  }
}, [answer]);
```

### 2. 完整的收藏功能实现

**核心特性**：
- ✅ **防重机制**：防止重复点击
- ✅ **防抖机制**：限制点击频率（500ms）
- ✅ **乐观更新**：立即更新UI，提升用户体验
- ✅ **状态同步**：同时更新收藏状态和收藏数量
- ✅ **错误回滚**：API失败时回滚UI状态
- ✅ **用户反馈**：显示成功/失败提示
- ✅ **API集成**：正确调用收藏/取消收藏接口

### 3. API接口调用

**收藏接口**：
```javascript
// 收藏
response = await answerApi.collectAnswer(answer.id);

// 取消收藏  
response = await answerApi.uncollectAnswer(answer.id);
```

**API端点配置**：
- 收藏：`POST /app/content/answer/:id/collect`
- 取消收藏：`POST /app/content/answer/:id/uncollect`

### 4. 数据同步机制

**乐观更新**：
```javascript
// 立即更新UI状态
setAnswerBookmarked(newBookmarked);

// 同时更新answerData中的收藏状态和数量
const currentCount = answerData.bookmarkCount || 0;
const newCount = newBookmarked ? currentCount + 1 : Math.max(0, currentCount - 1);

setAnswerData(prev => ({
  ...prev,
  isBookmarked: newBookmarked,
  bookmarked: newBookmarked,
  collected: newBookmarked,
  bookmarkCount: newCount,
  bookmark_count: newCount,
  bookmarks: newCount
}));
```

**API响应处理**：
```javascript
// 如果API返回了新的统计数据，使用API返回的数据
if (response.data.data) {
  const apiData = response.data.data;
  setAnswerData(prev => ({
    ...prev,
    bookmarkCount: apiData.bookmarkCount || prev.bookmarkCount,
    isBookmarked: apiData.isBookmarked !== undefined ? apiData.isBookmarked : newBookmarked
  }));
}
```

### 5. 错误处理和回滚

**状态回滚**：
```javascript
catch (error) {
  // 回滚UI状态
  setAnswerBookmarked(!answerBookmarked);
  
  // 回滚answerData中的状态
  if (answerData) {
    const originalCount = answerData.bookmarkCount || 0;
    const rolledBackCount = answerBookmarked ? Math.max(0, originalCount - 1) : originalCount + 1;
    
    setAnswerData(prev => ({
      ...prev,
      isBookmarked: answerBookmarked,
      bookmarkCount: rolledBackCount
    }));
  }
  
  // 显示错误提示
  toast.error(error.message || '网络错误，请重试');
}
```
