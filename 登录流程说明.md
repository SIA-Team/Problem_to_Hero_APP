## 详细说明

### 情况 1：有 token（用户已登录）

**场景**：用户之前已经登录过，应用保存了 token

**流程**：
1. 检测到本地有 `authToken`
2. 调用 `/app/user/auth/token-login` 接口验证 token
3. **成功**（`code === 200`）：
   - 设置 `isLoggedIn = true`
   - 进入应用主界面
   - 后台加载用户信息（使用缓存策略）
4. **失败**（token 过期）：
   - 清除旧的 token 和用户信息
   - 检查是否有设备指纹
     - **有设备指纹**：尝试用设备指纹重新注册
       - 成功：进入应用
       - 失败：显示登录页
     - **无设备指纹**：显示登录页

### 情况 2：无 token，有设备指纹（用户已注册但退出）

**场景**：用户之前注册过，但主动退出登录了

**流程**：
1. 检测到无 `authToken`，但有 `deviceFingerprint`
2. 判断为用户主动退出登录
3. 显示登录页面
4. 用户可以：
   - 输入用户名和密码登录
   - 点击"使用设备登录"重新注册

### 情况 3：无 token，无设备指纹（首次使用）

**场景**：用户首次安装应用

**流程**：
1. 检测到无 `authToken`，也无 `deviceFingerprint`
2. 判断为首次使用
3. 生成设备指纹
4. 调用 `/app/user/auth/register` 接口自动注册
5. **成功**：
   - 保存 token 到 `AsyncStorage`
   - 保存用户信息到 `AsyncStorage`
   - 保存设备指纹到 `AsyncStorage`
   - 设置 `isLoggedIn = true`
   - 进入应用主界面
   - 显示欢迎提示（用户名和默认密码）
6. **失败**（重试 3 次后仍失败）：
   - 显示登录页面
   - 提示用户点击"使用设备登录"重试

## 数据存储

### AsyncStorage 存储的数据

| 键名 | 说明 | 何时保存 | 何时清除 |
|------|------|---------|---------|
| `authToken` | 认证 token | 注册/登录成功 | 退出登录/token 过期 |
| `userInfo` | 用户基本信息 | 注册/登录成功 | 退出登录 |
| `deviceFingerprint` | 设备指纹 | 首次注册 | 永不清除（除非卸载应用） |
| `userProfileCache` | 用户详细信息缓存 | 获取用户信息 | 退出登录 |
| `usernameLastModified` | 用户名上次修改时间 | 修改用户名 | 退出登录 |

### 为什么保留设备指纹？

设备指纹不会被清除，即使用户退出登录。原因：
- 用于判断用户是否已注册过
- 避免重复注册
- 用户退出后可以用设备指纹重新登录

## Token 自动登录的优势

### 用户体验

- ✅ 用户无需每次都输入用户名和密码
- ✅ 应用启动即可使用
- ✅ Token 有效期内（720 小时 = 30 天）保持登录状态

### 安全性

- ✅ Token 有过期时间，过期后需要重新登录
- ✅ Token 存储在本地，不会泄露
- ✅ 服务器可以主动使 token 失效

## Mock 环境下的行为

### 首次启动

1. 生成设备指纹
2. 调用 Mock 注册接口
3. Mock 返回随机用户数据（用户名、昵称等）
4. 保存 token 和用户信息
5. 进入应用

### 再次启动

1. 检测到本地有 token
2. 调用 Mock token 登录接口
3. Mock 返回新的随机用户数据（与首次不同）
4. 更新本地用户信息
5. 进入应用

**注意**：Mock 环境下，每次 token 登录都会返回新的随机数据，这是正常的。真实环境下，token 登录会返回同一个用户的数据。

### 缓存优化

为了避免 Mock 环境下用户信息频繁变化，已优化缓存策略：
- Mock 环境：24 小时缓存
- 真实环境：30 分钟缓存

## 退出登录流程

### 用户主动退出

1. 点击"退出登录"按钮
2. 调用 `/app/user/auth/logout` 接口
3. 清除本地数据：
   - ✅ 清除 `authToken`
   - ✅ 清除 `userInfo`
   - ✅ 清除 `userProfileCache`
   - ❌ 保留 `deviceFingerprint`（用于判断已注册）
4. 返回登录页面

### Token 过期

1. Token 自动登录失败
2. 尝试用设备指纹重新注册
3. 成功：进入应用
4. 失败：显示登录页面

## 常见问题

### Q1: 为什么退出登录后还能看到设备指纹？

A: 设备指纹用于判断用户是否已注册过，不会被清除。这样可以：
- 避免重复注册
- 用户退出后可以用设备指纹重新登录

### Q2: Token 有效期是多久？

A: 根据后端返回的 `expiresIn` 字段，通常是 720 小时（30 天）。

### Q3: Mock 环境下为什么用户信息会变化？

A: Mock 环境每次请求都返回随机数据。已通过缓存优化，24 小时内保持一致。

### Q4: 如何测试 token 过期场景？

A: 
1. 手动删除 `authToken`：`await AsyncStorage.removeItem('authToken')`
2. 或者等待 token 真正过期（30 天后）
3. 或者在 Apifox Mock 中配置 token 登录接口返回失败

### Q5: 首次注册失败怎么办？

A: 应用会显示登录页面，用户可以：
- 点击"使用设备登录"重试
- 或者输入用户名和密码登录（如果已有账号）
