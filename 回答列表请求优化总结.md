# 回答列表请求优化总结

## 优化目标
减少服务器压力，避免每次切换排序方式（精选/最新）都重复请求接口。

## 优化策略
实现**分排序方式缓存机制**，为每个排序方式（featured/newest）分别缓存数据。

## 核心改进

### 1. 状态结构优化
```javascript
// 优化前：单一状态
const [answersList, setAnswersList] = useState([]);
const [answersTotal, setAnswersTotal] = useState(0);
const [answersPageNum, setAnswersPageNum] = useState(1);
const [answersHasMore, setAnswersHasMore] = useState(true);

// 优化后：分排序缓存
const [answersCache, setAnswersCache] = useState({
  featured: { list: [], total: 0, pageNum: 1, hasMore: true, loaded: false },
  newest: { list: [], total: 0, pageNum: 1, hasMore: true, loaded: false }

// 动态获取当前排序的数据
const currentAnswersData = answersCache[answersSortBy];

### 2. 智能加载逻辑

const loadAnswersList = async (questionId, isRefresh = false, sortBy = answersSortBy) => {
  // 检查缓存，避免重复请求

  if (!isRefresh && cacheData.loaded && cacheData.list.length > 0) {
    console.log(`📋 使用缓存数据: sortBy=${sortBy}`);
    return; // 直接返回，不发起请求
  }
  
  // 只有在没有缓存或需要刷新时才请求接口
  // ... 请求逻辑

```

### 3. 排序切换优化
```javascript
const handleAnswersSortChange = (newSortBy) => {
  if (newSortBy !== answersSortBy) {
    setAnswersSortBy(newSortBy);
    
    const newSortCache = answersCache[newSortBy];
    if (!newSortCache.loaded || newSortCache.list.length === 0) {
      // 没有缓存，需要加载
      loadAnswersList(questionId, true, newSortBy);
    } else {
      // 有缓存，直接使用，不发起请求
      console.log(`📋 使用缓存: ${newSortBy}`);
    }
  }
};
```

### 4. UI筛选条适配

- **补充问题**：featured(精选) / latest(最新)
- **回答列表**：featured(精选) / newest(最新)
- **评论**：保持原有逻辑

## 优化效果

### 请求次数对比
**优化前**：
- 初次加载：1次请求
- 切换到"最新"：1次请求
- 切换回"精选"：1次请求（重复）
- 再次切换到"最新"：1次请求（重复）
- **总计**：4次请求

**优化后**：
- 初次加载：1次请求
- 切换到"最新"：1次请求
- 切换回"精选"：0次请求（使用缓存）
- 再次切换到"最新"：0次请求（使用缓存）
- **总计**：2次请求

### 性能提升
- **减少50%的API请求**
- **提升切换响应速度**（缓存数据瞬间显示）
- **减少服务器压力**
- **改善用户体验**（无加载等待）

## 缓存策略

### 缓存生命周期
- **页面级缓存**：在当前问题详情页面内有效
- **自动失效**：离开页面后清空
- **手动刷新**：下拉刷新会更新对应排序的缓存

### 缓存数据结构
```javascript
{
  featured: {
    list: [],        // 回答列表数据
    total: 0,        // 总数量
    pageNum: 1,      // 下一页页码
    hasMore: true,   // 是否还有更多数据
    loaded: false    // 是否已加载过
  },
  newest: { ... }    // 同样的结构
}
```

### 分页支持
