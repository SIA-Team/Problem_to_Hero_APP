# 标题字段问题分析和解决方案

## 问题描述
用户反馈首页显示的问题列表中，标题显示的是描述内容而不是简洁的问题标题。

## 问题分析

### 1. 代码逻辑检查
经过检查，前端代码逻辑是正确的：

- **API层面**：`src/services/api/questionApi.js` 正确请求问题列表接口
- **数据转换层面**：`src/utils/dataLoader.js` 正确使用 `item.title` 字段
- **UI层面**：`src/screens/HomeScreen.js` 正确渲染 `item.title` 字段
- **问题详情页面**：正确区分 `title`（标题）和 `description`（描述）

### 2. 可能的原因
1. **后端数据问题**：后端可能把描述内容放在了 `title` 字段中
2. **字段混淆**：后端可能把 `title` 和 `description` 字段的内容搞反了
3. **数据格式问题**：后端返回的 `title` 字段包含了完整的问题描述

## 解决方案

### 方案1：前端智能处理（已实现）
在 `src/utils/dataLoader.js` 中添加了 `processTitle` 函数，对过长的标题进行智能处理：

```javascript
const processTitle = (title) => {
  if (!title || typeof title !== 'string') {
    return '无标题';
  }
  
  title = title.trim();
  
  // 如果标题过长（超过50个字符），可能包含了描述内容
  if (title.length > 50) {
    // 1. 如果包含问号，截取到第一个问号
    const questionMarkIndex = title.indexOf('？') !== -1 ? title.indexOf('？') : title.indexOf('?');
    if (questionMarkIndex !== -1 && questionMarkIndex < 50) {
      return title.substring(0, questionMarkIndex + 1);
    }
    
    // 2. 如果包含句号，截取到第一个句号
    const periodIndex = title.indexOf('。') !== -1 ? title.indexOf('。') : title.indexOf('.');
    if (periodIndex !== -1 && periodIndex < 50) {
      return title.substring(0, periodIndex + 1);
    }
    
    // 3. 截取前40个字符并加省略号
    return title.substring(0, 40) + '...';
  }
  
  return title;
};
```

### 方案2：后端修正（推荐）
联系后端开发人员，确保：
1. `title` 字段返回简洁的问题标题（建议20-50字符）
2. `description` 字段返回详细的问题描述
3. 字段内容不要搞混

## 调试工具
创建了 `debug-title-field.js` 调试脚本，可以帮助分析标题字段的问题。

## 验证方法
1. 查看控制台日志，检查后端返回的原始数据
2. 观察是否有"标题过长"的警告信息
3. 确认首页显示的标题是否合理

## 建议
1. **短期解决**：使用前端智能处理方案
2. **长期解决**：后端规范化数据格式
3. **数据规范**：建议标题长度控制在20-50字符内
