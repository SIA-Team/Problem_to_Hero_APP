# 收藏接口500错误修复总结

## 问题描述
收藏回答功能返回500错误，错误信息显示："No static resource app/content/answer/1/collect"

## 错误分析

### 1. 错误信息解读
```
API Response: {"code": 500, "msg": "No static resource app/content/answer/1/collect."}
```

这个错误信息表明：
- 服务器将API请求当作了静态资源请求
- URL路径构建是正确的：`/app/content/answer/1/collect`
- 问题不在URL构建，而在请求的处理方式

### 2. 根本原因分析
通过代码检查发现，不同的API使用了不同的HTTP客户端：

```javascript
// 获取回答列表 - 使用 contentApiClient ✅
return contentApiClient.get(url, { params: requestParams });

// 发布回答 - 使用 contentApiClient ✅  
return contentApiClient.post(url, answerCreateRequest);

// 收藏回答 - 使用 apiClient ❌
return apiClient.post(url);
```

**问题根源**：收藏接口使用了错误的API客户端！

### 3. API客户端差异
项目中存在两个不同的API客户端：

#### apiClient
- 用于通用API请求
- 可能配置了不同的baseURL或请求头
- 不适用于内容相关的API

#### contentApiClient  
- 专门用于内容相关的API请求
- 配置了正确的baseURL和认证信息
- 适用于问题、回答、补充等内容API

## 解决方案

### 修复前
```javascript
collectAnswer: (answerId) => {
  const url = replaceUrlParams(API_ENDPOINTS.ANSWER.COLLECT, { id: answerId });
  return apiClient.post(url); // ❌ 错误的客户端
},
```

### 修复后
```javascript
collectAnswer: (answerId) => {
  const url = replaceUrlParams(API_ENDPOINTS.ANSWER.COLLECT, { id: answerId });
  
  console.log('📤 收藏/取消收藏回答:');
  console.log('  原始端点:', API_ENDPOINTS.ANSWER.COLLECT);
  console.log('  answerId:', answerId);
  console.log('  构建后的URL:', url);
  console.log('  请求方法: POST (使用 contentApiClient)');
  
  return contentApiClient.post(url); // ✅ 正确的客户端
},
```
