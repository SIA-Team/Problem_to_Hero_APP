# 发布问题选择类别支持自定义二级类别需求说明

## 1. 需求背景
用户在发布问题时，现有二级分类为固定选项，无法覆盖新出现或长尾场景，导致分类表达不足、检索命中差。

## 2. 需求目标
- 在“发布问题 > 选择问题类别 > 二级分类”阶段，支持用户输入并选择自定义二级类别。
- 自定义二级类别数据上送后台管理，用于后续审核与入库。
- 在类别未入库前，保证问题可被正确展示与检索，但不作为正式筛选项暴露。

## 3. 功能范围
### 3.1 包含范围
- 二级分类弹层新增自定义输入与选择能力。
- 发布/存草稿时携带自定义类别扩展字段。
- 未入库与已入库两种状态的展示/筛选规则定义。

### 3.2 不包含范围
- 后台类目审核流程详细设计。
- 搜索引擎底层改造（仅定义搜索字段要求）。

## 4. 业务规则
1. 用户可在二级分类页输入自定义类别名并选择。
2. 自定义类别需绑定当前一级大类。
3. 若输入名称与当前一级下已有二级类别同名（忽略首尾空格、大小写），直接命中现有类别，不重复创建。
4. 自定义类别“未入库”时：
- 问题详情/列表可显示自定义类别名称。
- 频道筛选与分类筛选中不新增该二级筛选项。
- 仅可在对应一级大类下看到该问题数据。
- 搜索命中自定义类别关键词时可检索到该问题。
5. 自定义类别“已入库”后：
- 作为正式二级类别出现在频道与筛选项中。
- 可按该二级类别直接筛选。

## 5. 发布与草稿数据要求
### 5.1 通用字段
- `categoryId`
- `categoryName`

### 5.2 自定义类别扩展字段（建议）
- `customCategoryName`
- `customCategoryParentId`
- `customCategoryParentName`
- `customCategoryLevel`（固定为 2）
- `customCategoryPending`（未入库标记，1=待入库）

### 5.3 categoryId 规则
- 标准二级类别：`categoryId = 二级类别ID`
- 自定义二级类别未入库：`categoryId = 一级类别ID`（用于在一级维度归类展示）

## 6. 交互流程
1. 用户选择一级类别。
2. 进入二级类别页，可搜索已有二级类别。
3. 用户可输入自定义二级类别并点击“添加/确认”。
4. 客户端完成同名校验：
- 命中已有类别 -> 选中已有类别。
- 未命中 -> 生成自定义类别对象并选中。
5. 用户发布或存草稿时提交分类与自定义扩展字段。
6. 后台管理接收待入库类别并处理审核入库。

## 7. 展示与检索规则
1. 问题卡片/详情显示：展示 `categoryName`（可为自定义名）。
2. 一级筛选：可筛到该自定义类别问题（因归属一级 `categoryId`）。
3. 二级筛选：未入库前不可见；入库后可见并可筛选。
4. 搜索：需支持基于问题中的自定义类别名称字段检索。

## 8. 异常与边界
1. 输入为空或纯空格：禁止添加并提示。
2. 输入过长/非法字符：按统一输入规范拦截。
3. 网络失败：发布失败需明确提示，不丢失本地输入。
4. 草稿回显：自定义类别应可完整回显并再次编辑。

## 9. 验收标准
1. 二级分类页可成功添加并选中自定义类别。
2. 发布成功后，问题显示自定义类别名称。
3. 未入库前：
- 二级筛选列表不出现该类别。
- 在对应一级大类下可见问题。
- 搜索自定义关键词可命中问题。
4. 入库后：二级筛选中出现该类别并可筛选到对应问题。

## 10. 埋点建议
- 自定义类别输入次数、成功选择次数。
- 发布中携带 `customCategoryPending=1` 的请求量。
- 未入库自定义类别的搜索命中率。
- 入库后该类别筛选使用率。
