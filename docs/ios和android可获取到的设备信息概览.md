## iOS 设备信息

### 1. IDFV (Identifier for Vendor) ⭐⭐⭐⭐⭐
```

| 属性 | 说明 |
|------|------|
| **稳定性** | ⭐⭐⭐⭐⭐ 非常稳定 |
| **卸载重装** | ✅ 保持不变 |
| **系统升级** | ✅ 保持不变 |
| **恢复出厂** | ❌ 会改变 |
| **设备更换** | ❌ 会改变 |
| **特殊情况** | ⚠️ 用户删除该开发者的所有应用后会重置 |
| **适用版本** | iOS 6.0+ |
| **隐私合规** | ✅ Apple 推荐，符合隐私政策 |
| **唯一性** | ✅ 每个设备对每个开发者唯一 |

**变化条件**：
- 用户删除该开发者账号下的所有应用
- 设备恢复出厂设置
- 更换设备

---

### 2. IDFA (Identifier for Advertisers)

| 属性 | 说明 |
|------|------|
| **稳定性** | ⭐⭐ 不稳定 |
| **需要权限** | ✅ iOS 14.5+ 需要 ATT 授权 |
| **用户可重置** | ✅ 用户可在设置中重置 |
| **卸载重装** | ✅ 保持不变（如果未重置） |
| **适用版本** | iOS 6.0+ |
| **隐私合规** | ⚠️ 需要明确告知用户用途 |
| **授权率** | ⚠️ iOS 14.5+ 后授权率很低（约15-30%） |

**变化条件**：
- 用户在设置中重置广告标识符
- 用户拒绝 ATT 授权（返回全0）
- 恢复出厂设置

- 违反隐私政策

## 当前项目实现分析

### 指纹组成

操作系统 (ios/android)
系统版本 (17.2.1)
品牌 (Apple/Samsung)
型号名称 (iPhone 14 Pro)
型号ID (iPhone15,2)
屏幕分辨率 (1170x2532)
时区 (Asia/Shanghai)

### 特点

#### ✅ 优点

1. **隐私友好**
   - 不涉及个人信息
   - 不需要特殊权限
   - 符合 GDPR、CCPA 等隐私法规

2. **跨平台一致**
   - iOS 和 Android 使用相同逻辑
   - 代码简单易维护

3. **卸载重装后保持**
   - 只要设备硬件不变，指纹不变
   - 适合防止重复注册

4. **无需网络请求**
   - 本地生成，速度快
   - 离线可用

#### ⚠️ 弊端

1. **唯一性不足**
   - 同型号、同系统版本、同时区的设备会产生相同指纹
   - 例如：两台新买的 iPhone 14 Pro（iOS 17.2.1，中国时区）= 相同指纹
   - **碰撞概率**：热门机型约 1-5%

2. **系统升级后改变**
   - 包含 `osVersion`，系统升级后指纹会变
   - 用户升级 iOS 17.2.1 → 17.3 后，指纹改变
   - **影响**：用户可能被识别为新设备

3. **时区变化影响**
   - 用户旅行或更改时区设置后，指纹改变
   - **影响**：国际用户体验差

4. **无法区分同型号设备**
   - 家庭成员使用相同型号手机，指纹可能相同
   - **影响**：多账号检测失效

### 实际案例分析

#### 案例 1：指纹碰撞
```
用户 A：iPhone 14 Pro, iOS 17.2.1, Asia/Shanghai
用户 B：iPhone 14 Pro, iOS 17.2.1, Asia/Shanghai
结果：相同指纹 ❌
```

#### 案例 2：系统升级
```
用户 A 升级前：iPhone 14 Pro, iOS 17.2.1
指纹：a1b2c3d4e5f6g7h8

用户 A 升级后：iPhone 14 Pro, iOS 17.3.0
指纹：x9y8z7w6v5u4t3s2

结果：被识别为新设备 ❌
```

#### 案例 3：时区变化
```
用户 A 在中国：Asia/Shanghai
指纹：a1b2c3d4e5f6g7h8

用户 A 去美国：America/New_York
指纹：m1n2o3p4q5r6s7t8

结果：被识别为新设备 ❌
```

### 改进建议

1. **移除易变信息**
   ```javascript
   // ❌ 不推荐包含
   info.platform.osVersion,  // 系统升级会变
   info.locale.timezone,     // 时区变化会变
   
   // ✅ 推荐保留
   info.platform.os,
   info.device.brand,
   info.device.modelName,
   info.device.modelId,
   screenResolution,
   ```

2. **结合平台特定 ID**
   - iOS: 优先使用 IDFV
   - Android: 优先使用 Android ID
   - 指纹作为降级方案

## 常见问题

### Q1: 为什么不推荐使用 IMEI？
A: Android 10+ 已禁止第三方应用获取 IMEI，且违反隐私政策。

### Q2: IDFV 什么时候会改变？
A: 用户删除该开发者的所有应用后，IDFV 会重置。

### Q3: 设备指纹的碰撞率有多高？
A: 当前实现约 1-5%（热门机型），改进后可降至 0.1-0.5%。

### Q4: 如何处理标识符变化？
A: 使用多标识符策略，后端支持标识符迁移和关联。
