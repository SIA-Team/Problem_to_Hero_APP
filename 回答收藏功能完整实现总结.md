# 回答收藏功能完整实现总结

## 功能概述

在回答详情页面底部实现了收藏和取消收藏功能，包含完整的防重和防抖机制，确保不会对服务器造成恶意点击和压力。

## 接口信息

### 收藏回答
- **接口地址**: `/app/content/answer/{id}/collect`
- **请求方式**: POST
- **请求参数**: `id`（回答ID，int类型，必传）
- **返回数据**: `{"code": 200, "msg": "", "data": {}}`

### 取消收藏回答
- **接口地址**: `/app/content/answer/{id}/uncollect`
- **请求方式**: POST
- **请求参数**: `id`（回答ID，int类型，必传）
- **返回数据**: `{"code": 200, "msg": "", "data": {}}`

## 核心代码

### 收藏处理函数
```javascript
const handleAnswerBookmark = async () => {
  // 防重检查
  if (isCollectLoading) return;
  
  // 防抖检查
  const now = Date.now();
  if (now - lastCollectTimeRef.current < 500) return;
  lastCollectTimeRef.current = now;
  
  // 防抖延迟执行
  collectTimeoutRef.current = setTimeout(async () => {
    try {
      setIsCollectLoading(true);
      
      // 乐观更新
      const newBookmarked = !answerBookmarked;
      setAnswerBookmarked(newBookmarked);
      
      // 调用接口
      const response = newBookmarked 
        ? await answerApi.collectAnswer(answer.id)
        : await answerApi.uncollectAnswer(answer.id);
      
      if (response?.data?.code === 200) {
        toast.success(newBookmarked ? '收藏成功' : '取消收藏成功');
      } else {
        throw new Error(response?.data?.msg || '操作失败');
      }
    } catch (error) {
      // 回滚UI状态
      setAnswerBookmarked(!answerBookmarked);
      toast.error(error.message || '网络错误，请重试');
    } finally {
      setIsCollectLoading(false);
    }
  }, 300);
};
```

### UI组件
```javascript
<TouchableOpacity
```

## 防护机制总结

### 1. 防重复请求
- 使用 `isCollectLoading` 状态标记
- 请求进行中时禁用按钮和忽略点击

### 2. 防频繁点击
- 500ms内只允许一次有效点击
- 使用 `lastCollectTimeRef` 记录上次点击时间

### 3. 防抖延迟
- 300ms延迟执行，避免误触
- 使用 `collectTimeoutRef` 管理定时器

### 4. 资源清理
- 组件卸载时清理定时器
- 避免内存泄漏
