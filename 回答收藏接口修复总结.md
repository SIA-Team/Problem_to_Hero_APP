# 回答收藏接口修复总结

## 问题描述

回答列表的收藏功能需要使用正确的接口：
- 收藏：`/app/content/answer/{id}/collect` (POST)
- 取消收藏：`/app/content/answer/{id}/uncollect` (POST)

## 修改内容

### 1. API 配置文件 (`src/config/api.js`)

添加取消收藏端点：

```javascript
ANSWER: {
  LIST: '/app/content/answer/question/:questionId/list',
  PUBLISH: '/app/content/answer/question/:questionId',
  COLLECT: '/app/content/answer/:id/collect',      // 收藏回答
  UNCOLLECT: '/app/content/answer/:id/uncollect',  // 取消收藏回答 ← 新增
  LIKE: '/app/content/answer/:id/like',
  ADOPT: '/app/content/answer/:id/adopt',
},
```

### 2. Answer API (`src/services/api/answerApi.js`)

**修改前**：
- 只有一个 `collectAnswer` 方法
- 需要传入 `questionId` 参数
- 尝试多种请求格式

**修改后**：
- 分离为两个独立方法：
  - `collectAnswer(answerId)` - 收藏回答
  - `uncollectAnswer(answerId)` - 取消收藏回答
- 只需要 `answerId` 参数
- 简化请求逻辑

```javascript
/**
 * 收藏回答
 * @param {string} answerId - 回答ID
 * @returns {Promise<Object>}
 */
collectAnswer: async (answerId) => {
  const url = replaceUrlParams(API_ENDPOINTS.ANSWER.COLLECT, { id: answerId });
  return contentApiClient.post(url);
},

/**
 * 取消收藏回答
 * @param {string} answerId - 回答ID
 * @returns {Promise<Object>}
 */
uncollectAnswer: async (answerId) => {
  const url = replaceUrlParams(API_ENDPOINTS.ANSWER.UNCOLLECT, { id: answerId });
  return contentApiClient.post(url);
},
```

### 3. 问题详情页面 (`src/screens/QuestionDetailScreen.js`)

**修改前**：
```javascript
const response = await answerApi.collectAnswer(answerId, questionId);
```

**修改后**：
```javascript
// 根据当前状态调用不同的接口
const response = newState 
  ? await answerApi.collectAnswer(answerId)
  : await answerApi.uncollectAnswer(answerId);
```

**优化点**：
1. 移除了 `questionId` 参数（不需要）
2. 根据目标状态调用对应接口
3. 简化了逻辑，更清晰

## 接口规范

### 收藏回答

**请求**：
```
POST /app/content/answer/{id}/collect
```

**参数**：
- `id` (路径参数, int): 回答ID

**响应**：
```json
{
  "code": 200,
  "msg": "",
  "data": {}
}
```

### 取消收藏回答

**请求**：
```
POST /app/content/answer/{id}/uncollect
```

**参数**：
- `id` (路径参数, int): 回答ID

**响应**：
```json
{
  "code": 200,
  "msg": "",
  "data": {}
}
```

## 功能特点

### 1. 即时反馈
- 点击后立即更新 UI
- 不等待服务器响应
- 提供流畅的用户体验

### 2. 防抖机制
- 300ms 防抖延迟
- 避免重复请求
- 优化网络性能

### 3. 错误处理
- 请求失败时回滚 UI 状态
- 显示友好的错误提示
- 记录详细的日志

### 4. 请求去重
- 使用 `answerCollectLoading` 状态
- 防止同一回答的并发请求
- 确保数据一致性
