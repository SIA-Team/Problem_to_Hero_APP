# 回答详情页数据同步完整实现

## 📋 需求回顾

用户在回答详情页取消对当前回答的收藏，实际上在问题详情页中的回答列表下的当前的那一条回答也应该是取消收藏的。意思就是问题详情页中回答列表下的每一个回答的点赞/取消点赞、收藏/取消收藏、踩/取消踩这些数字是要同步到回答详情页中的。

## ✅ 已完成的工作

### 1. 修复回答详情页收藏功能

**问题**：回答详情页的收藏按钮提示"操作失败"

**原因**：响应数据结构理解错误
- `contentApiClient` 的响应拦截器已经返回了 `response.data`
- 代码错误地访问了 `response.data.code`（实际是 `response.data.data.code`，不存在）

**修复**：
```javascript
// 修改前
const responseData = response?.data;
const responseCode = responseData?.code;

// 修改后
const responseCode = response?.code;
const responseMsg = response?.msg;
const responseDataField = response?.data;
```

**文件**：`src/screens/AnswerDetailScreen.js`

### 2. 实现跨页面数据同步

**方案**：React Navigation Params + Focus Listener

**优势**：
- ✅ 简单直接，无需额外依赖
- ✅ 数据流清晰，易于调试
- ✅ 性能开销小
- ✅ 支持多排序方式缓存

#### 2.1 问题详情页 (QuestionDetailScreen.js)

**添加 Focus Listener**：
```javascript
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    // 检查缓存过期...
    
    // 检查是否有从回答详情页返回的更新数据
    const updatedAnswer = route?.params?.updatedAnswer;
    if (updatedAnswer) {
      console.log('🔄 检测到回答数据更新，同步到列表:', updatedAnswer.id);
      syncAnswerData(updatedAnswer);
      navigation.setParams({ updatedAnswer: undefined });
    }
  });
  return unsubscribe;
}, [navigation, answersCache, route?.params?.id, route?.params?.updatedAnswer]);
```

**实现 syncAnswerData 函数**：
```javascript
const syncAnswerData = (updatedAnswer) => {
  if (!updatedAnswer || !updatedAnswer.id) return;
  
  // 更新所有排序方式的缓存
  setAnswersCache(prevCache => {
    const newCache = { ...prevCache };
    
    Object.keys(newCache).forEach(sortBy => {
      const cacheData = newCache[sortBy];
      const answerIndex = cacheData.list.findIndex(a => a.id === updatedAnswer.id);
      
      if (answerIndex !== -1) {
        const updatedList = [...cacheData.list];
        updatedList[answerIndex] = {
          ...updatedList[answerIndex],
          // 用户状态
          isLiked: updatedAnswer.isLiked,
          isBookmarked: updatedAnswer.isBookmarked,
          isDisliked: updatedAnswer.isDisliked,
          // 统计数据
          likeCount: updatedAnswer.likeCount,
          bookmarkCount: updatedAnswer.bookmarkCount,
          dislikeCount: updatedAnswer.dislikeCount,
          // ... 其他兼容字段
        };
        
        newCache[sortBy] = {
          ...cacheData,
          list: updatedList
        };
      }
    });
    
    return newCache;
  });
  
  // 同步本地状态
  setAnswerLiked(prev => ({ ...prev, [updatedAnswer.id]: updatedAnswer.isLiked }));
  setAnswerBookmarked(prev => ({ ...prev, [updatedAnswer.id]: updatedAnswer.isBookmarked }));
  setAnswerDisliked(prev => ({ ...prev, [updatedAnswer.id]: updatedAnswer.isDisliked }));
};
```

#### 2.2 回答详情页 (AnswerDetailScreen.js)

**实现 syncDataToQuestionDetail 函数**：

```javascript
const syncDataToQuestionDetail = () => {
  if (!answerData || !navigation) return;
  
  // 获取导航栈中的问题详情页
  const routes = navigation.getState()?.routes;
  const questionDetailRoute = routes?.find(r => r.name === 'QuestionDetail');
  
  if (questionDetailRoute) {
    // 构建更新的回答数据
    const updatedAnswer = {
      id: answerData.id,
      isLiked: answerLiked,
      isBookmarked: answerBookmarked,
      isDisliked: answerDisliked,
      likeCount: answerData.likeCount || 0,
      bookmarkCount: answerData.bookmarkCount || 0,
      dislikeCount: answerData.dislikeCount || 0,
      // ... 其他兼容字段
    };
    
    // 通过 navigation.navigate 传递数据
    navigation.navigate('QuestionDetail', {
      ...questionDetailRoute.params,
      updatedAnswer: updatedAnswer
    });
  }
};
```

## 💡 经验总结

### 技术选型
- ✅ 对于简单的跨页面数据同步，React Navigation Params 足够
- ✅ 无需过早引入复杂的状态管理库
- ✅ 保持代码简单，易于维护

### 最佳实践
1. **数据一致性**：同时更新所有相关的缓存和状态
2. **错误处理**：操作失败时回滚状态
3. **日志记录**：添加详细的日志便于调试
4. **测试覆盖**：编写测试脚本验证逻辑

### 注意事项
1. **避免重复处理**：处理完数据后立即清除 params
2. **兼容性**：支持多种字段命名格式
3. **性能**：只更新变化的数据，不重新加载整个列表
4. **用户体验**：乐观更新 + 错误回滚
