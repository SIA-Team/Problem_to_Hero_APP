## 🔒 版本特定限制和配置

### 1. Android 6.0 (API 23) - 运行时权限

**影响**：需要在运行时请求权限

**已配置的权限**：
```xml
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.CAMERA"/>
<uses-permission android:name="android.permission.RECORD_AUDIO"/>
<uses-permission android:name="android.permission.VIBRATE"/>
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW"/>
```

**应用中的处理**：
- ✅ 使用 `expo-image-picker` 自动处理相册权限
- ✅ 使用 `expo-camera` 自动处理相机权限
- ✅ 权限被拒绝时有友好提示

**无需额外配置**

---

### 2. Android 8.0 (API 26) - 通知渠道

**影响**：需要创建通知渠道才能显示通知

**当前状态**：应用暂未使用推送通知

**如果未来需要推送**：
```javascript
import * as Notifications from 'expo-notifications';

// 创建通知渠道
await Notifications.setNotificationChannelAsync('default', {
  name: 'Default',
  importance: Notifications.AndroidImportance.MAX,
});
```

**无需立即配置**

---

### 3. Android 9 (API 28) - 明文流量限制 ⚠️ 重要

**影响**：默认禁止 HTTP 明文流量，只允许 HTTPS

**问题**：你的服务器使用 HTTP (`http://123.144.100.10:30560`)

**已修复**：
```xml
<application
  ...
  android:usesCleartextTraffic="true">
```

**状态**：✅ 已配置

**建议**：长期应升级到 HTTPS

---

### 4. Android 10 (API 29) - 分区存储

**影响**：限制应用访问外部存储

**当前配置**：
```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
```

**应用中的处理**：
- ✅ 使用 `expo-image-picker` 访问相册（不需要直接访问文件系统）
- ✅ 使用 `expo-file-system` 访问应用私有目录
- ✅ 不直接访问外部存储的文件

**无需额外配置**

---

### 5. Android 11 (API 30) - 包可见性

**影响**：需要声明要查询的其他应用

**当前配置**：
```xml
<queries>
  <intent>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="https"/>
  </intent>
</queries>
```

**用途**：允许应用打开浏览器链接

**状态**：✅ 已配置

---

### 6. Android 12 (API 31) - 精确闹钟权限

**影响**：需要特殊权限才能设置精确闹钟

**当前状态**：应用不使用闹钟功能

**无需配置**

---

### 7. Android 13 (API 33) - 通知权限

**影响**：需要运行时请求通知权限

**当前状态**：应用暂未使用推送通知

**如果未来需要推送**：
```xml
<uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
```

**无需立即配置**

---

### 8. Android 13 (API 33) - 媒体权限细化

**影响**：存储权限被细分为图片、视频、音频

**新权限**：
```xml
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES"/>
<uses-permission android:name="android.permission.READ_MEDIA_VIDEO"/>
<uses-permission android:name="android.permission.READ_MEDIA_AUDIO"/>
```

**当前配置**：
```json
"permissions": [
  "CAMERA",
  "READ_EXTERNAL_STORAGE",
  "WRITE_EXTERNAL_STORAGE",
  "READ_MEDIA_IMAGES"
]
```

**状态**：✅ 已配置 `READ_MEDIA_IMAGES`

**应用中的处理**：
- ✅ `expo-image-picker` 自动处理新旧权限兼容

---

### 9. Android 14 (API 34) - 前台服务类型

**影响**：前台服务需要声明类型

**当前状态**：应用不使用前台服务

**无需配置**

---

### 10. Android 15 (API 35) - 目标版本

**影响**：最新的 Android 版本

**当前配置**：`targetSdkVersion = 35`

**状态**：✅ 已配置为目标版本

## 📊 权限使用说明

### 必需权限

| 权限 | 用途 | 请求时机 |
|-----|------|---------|
| INTERNET | 网络请求 | 安装时自动授予 |
| CAMERA | 拍照上传头像 | 使用相机时 |
| READ_MEDIA_IMAGES | 选择相册图片 | 选择图片时 |

### 可选权限

| 权限 | 用途 | 请求时机 |
|-----|------|---------|
| VIBRATE | 震动反馈 | 安装时自动授予 |
| RECORD_AUDIO | 录音（未来功能） | 暂未使用 |

### 已移除的权限

- ❌ `ACCESS_FINE_LOCATION` - 不需要定位
- ❌ `READ_PHONE_STATE` - 不需要读取手机状态
- ❌ `GET_ACCOUNTS` - 不需要访问账户

## 🔧 特殊配置说明

### 1. 明文流量配置（重要）

**文件**：`android/app/src/main/AndroidManifest.xml`

```xml
<application
  ...
  android:usesCleartextTraffic="true">
```

**用途**：允许 HTTP 连接（因为服务器使用 HTTP）

**安全建议**：
- ⚠️ 这是临时方案
- 🔒 应尽快升级到 HTTPS
- 📱 生产环境强烈建议使用 HTTPS

### 2. 状态栏配置

**文件**：`app.json`

```json
"androidStatusBar": {
  "backgroundColor": "#ffffff",
  "barStyle": "dark-content",
  "translucent": false
}
```

**效果**：白色状态栏，深色图标

### 3. 软键盘模式

**文件**：`app.json`

```json
"softwareKeyboardLayoutMode": "resize"
```

**效果**：键盘弹出时调整布局大小（不遮挡输入框）

## 🐛 已知问题和解决方案

### 问题 1: 生产版本无法自动注册

**症状**：首次安装显示登录页，不自动注册

**原因**：Android 9+ 禁止明文 HTTP 流量

**解决方案**：✅ 已添加 `android:usesCleartextTraffic="true"`

**状态**：已修复

---

### 问题 2: 相册权限在 Android 13+ 上失败

**症状**：无法选择图片

**原因**：Android 13 细化了媒体权限

**解决方案**：✅ 已添加 `READ_MEDIA_IMAGES` 权限

**状态**：已修复

## 📞 问题排查

### 如果用户报告无法使用

1. **确认 Android 版本**
   - 是否 >= Android 6.0？
   - 如果 < 6.0，不支持

2. **检查网络连接**

3. **检查权限**
   - 相机/相册权限是否授予？
   - 在系统设置中检查

4. **查看日志**
   - 使用 `adb logcat` 查看错误
