# 草稿编辑功能实现总结

## 功能描述
点击"我的草稿"列表中的草稿项，获取草稿完整数据并回显到发布问题页面进行编辑。

## 接口信息
- **接口地址**: `/app/content/question/{id}`
- **请求方式**: GET
- **请求参数**: id (int) - 草稿ID
- **返回数据**: 完整的草稿数据（包括标题、描述、分类、图片、话题、专家等）


**修改后：**
```javascript
const handleDraftPress = async (item) => {
  try {
    console.log('📝 点击草稿，ID:', item.id);
    
    // 关闭草稿列表弹窗
    setShowDraftsModal(false);
    
    // 调用接口获取草稿完整数据
    const response = await questionApi.getDraftDetail(item.id);
    
    if (response && response.code === 200 && response.data) {
      const draftData = response.data;
      
      // 跳转到发布页面，传递完整的草稿数据
      navigation.navigate('Publish', { draftData });
    } else {
      Alert.alert('获取草稿失败', response?.msg || '无法加载草稿数据');
    }
  } catch (error) {
    console.error('获取草稿详情失败:', error);
    Alert.alert('获取草稿失败', '网络错误，请稍后重试');
  }
};
```

### 3. 实现草稿数据回显

#### src/screens/PublishScreen.js

**接收路由参数：**
```javascript
export default function PublishScreen({ navigation, route }) {
  // 获取路由参数中的草稿数据
  const draftData = route?.params?.draftData;
  // ...
}
```

**添加回显逻辑：**
```javascript
// 回显草稿数据
useEffect(() => {
  if (draftData) {
    console.log('📝 回显草稿数据');
    
    // 基础信息
    if (draftData.title) setTitle(draftData.title);
    if (draftData.description) setContent(draftData.description);
    if (typeof draftData.type === 'number') setSelectedType(draftData.type);
    if (draftData.location) setLocation(draftData.location);
    if (typeof draftData.visibilityScope === 'number') setVisibility(draftData.visibilityScope);
    if (typeof draftData.isAnonymous === 'number') setIsAnonymous(draftData.isAnonymous === 1);
    if (typeof draftData.isPublicAnswer === 'number') setAnswerPublic(draftData.isPublicAnswer === 1);
    
    // 悬赏金额（分转元）
    if (draftData.bountyAmount && draftData.bountyAmount > 0) {
      const amount = (draftData.bountyAmount / 100).toString();
      if (draftData.type === 2) { // 定向问题
        setTargetedReward(amount);
      } else {
        setReward(amount);
      }
    }
    
    // 付费查看金额（分转元）
    if (draftData.payViewAmount && draftData.payViewAmount > 0) {
      setAnswerPaid(true);
      setAnswerPrice((draftData.payViewAmount / 100).toString());
    }
    
    // 图片
    if (draftData.imageUrls && Array.isArray(draftData.imageUrls)) {
      setImages(draftData.imageUrls);
    }
    
    // 话题
    if (draftData.topics && Array.isArray(draftData.topics)) {
      const topicNames = draftData.topics.map(t => `#${t.name}`);
      setSelectedTopics(topicNames);
      setAllTopics(topicNames);
      setCustomTopicNames(draftData.topics.map(t => t.name));
    }
    
    // 专家（定向问题）
    if (draftData.experts && Array.isArray(draftData.experts)) {
      setTargetedUsers(draftData.experts);
    }
    
    // 分类（延迟设置，等待分类数据加载）
    if (draftData.categoryId) {
      setTimeout(() => {
        // 查找对应的分类
        for (const level1 of level1Categories) {
          const level2List = level2CategoriesMap[level1.id] || [];
          const level2 = level2List.find(cat => cat.id === draftData.categoryId);
          if (level2) {
            setSelectedLevel1(level1);
            setSelectedLevel2(level2);
            break;
          }
        }
      }, 1000);
    }
  }
}, [draftData, level1Categories, level2CategoriesMap]);
```

## 数据回显字段映射

| 后端字段 | 前端状态 | 转换逻辑 |
|---------|---------|---------|
| title | title | 直接赋值 |
| description | content | 直接赋值 |
| type | selectedType | 直接赋值（0/1/2） |
| categoryId | selectedLevel1, selectedLevel2 | 查找对应的分类对象 |
| location | location | 直接赋值 |
| visibilityScope | visibility | 直接赋值（0/1/2） |
| isAnonymous | isAnonymous | 1转true，0转false |
| isPublicAnswer | answerPublic | 1转true，0转false |
| bountyAmount | reward / targetedReward | 分转元（除以100） |
| payViewAmount | answerPaid, answerPrice | 分转元，大于0时启用付费 |
| imageUrls | images | 直接赋值数组 |
| topics | selectedTopics, allTopics, customTopicNames | 转换为带#的格式 |
| experts | targetedUsers | 直接赋值数组 |

## 用户流程

1. **用户点击草稿**
   - 在"我的草稿"列表中点击某个草稿项

2. **获取草稿详情**
   - 调用 `/app/content/question/{id}` 接口
   - 获取完整的草稿数据

3. **跳转到发布页面**
   - 通过 navigation.navigate 传递草稿数据
   - 发布页面接收 route.params.draftData

4. **回显草稿数据**
   - useEffect 监听 draftData 变化
   - 将草稿数据填充到各个表单字段
   - 分类数据延迟设置（等待分类列表加载）

5. **编辑并发布**
   - 用户可以修改草稿内容
   - 点击发布按钮提交

## 注意事项

### 1. 金额转换
- 后端存储单位：分
- 前端显示单位：元
- 转换公式：元 = 分 / 100

### 2. 布尔值转换
- 后端：0/1
- 前端：false/true
- 转换：1 === 1 → true, 0 === 1 → false

### 3. 分类回显
- 需要等待分类数据加载完成
- 使用 setTimeout 延迟1秒设置
- 遍历一级分类和二级分类查找匹配项

### 4. 话题格式
- 后端：`{ id: 1, name: "人工智能" }`
- 前端显示：`#人工智能`
- 前端发送：`"人工智能"`（不含#）

### 5. 错误处理
- 接口调用失败时显示错误提示
- 网络错误时提示用户重试
- 数据格式异常时使用默认值
