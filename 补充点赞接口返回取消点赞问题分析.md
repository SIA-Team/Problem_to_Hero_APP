# 补充点赞接口返回"已取消点赞"问题分析

## 问题现象
用户点击补充列表的点赞按钮，明明是想点赞，但接口返回"已取消点赞"。

## 可能原因分析

### 1. 前端状态初始化问题（最可能）
**问题：** 前端没有正确初始化补充的点赞状态
- 后端数据库中该补充已被当前用户点赞过
- 但前端 `suppLiked` 状态为空 `{}`
- 用户看到的是未点赞状态（空心图标）
- 实际后端已点赞，点击时执行取消操作

### 2. 补充接口是Toggle模式
**特点：** 补充点赞接口 `/app/content/supplement/:id/like` 是切换模式
- 如果未点赞 → 执行点赞 → 返回 `{data: true}`
- 如果已点赞 → 执行取消 → 返回 `{data: false}`

### 3. 数据同步问题
可能在其他地方（如其他页面）点过赞，但当前页面状态没同步

## 解决方案

### 方案1：检查补充列表接口返回数据
确认补充列表接口是否返回每条补充的点赞状态：
```json
{
  "id": 456,
  "content": "补充内容",
  "likes": 10,
  "isLiked": true  // 关键字段
}
```

### 方案2：修复前端初始化（已实现）
在 `loadSupplementsList` 中初始化状态：
```javascript
newSupplements.forEach(item => {
  initialLikedState[item.id] = item.isLiked || false;
});
setSuppLiked(prev => ({ ...prev, ...initialLikedState }));
```
