# 回答数据跨页面同步实现总结

## 需求描述

用户在回答详情页对回答进行操作（点赞/收藏/踩）后，返回到问题详情页时，该回答在列表中的状态应该保持一致。

## 实现方案

采用 **React Navigation Params 传递 + Focus Listener** 的方案，无需引入额外的状态管理库。

### 方案优势

1. ✅ 简单直接，利用 React Navigation 原生能力
2. ✅ 无需引入 Redux/MobX 等状态管理库
3. ✅ 数据流清晰，易于调试
4. ✅ 性能开销小

### 数据流向

```
回答详情页 (AnswerDetailScreen)
    ↓ 用户操作（收藏/点赞/踩）
    ↓ API 调用成功
    ↓ 更新本地状态
    ↓ 调用 syncDataToQuestionDetail()
    ↓ navigation.navigate('QuestionDetail', { updatedAnswer })
    ↓
问题详情页 (QuestionDetailScreen)
    ↓ focus listener 触发
    ↓ 检测到 route.params.updatedAnswer
    ↓ 调用 syncAnswerData(updatedAnswer)
    ↓ 更新缓存中的回答数据
    ↓ 更新本地状态
    ✅ UI 自动刷新
```

## 实现细节

### 1. 问题详情页 (QuestionDetailScreen.js)

#### 1.1 添加 Focus Listener

```javascript
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    const questionId = route?.params?.id;
    if (!questionId) return;

    // 检查缓存过期...
    
    // 🔄 检查是否有从回答详情页返回的更新数据
    const updatedAnswer = route?.params?.updatedAnswer;
    if (updatedAnswer) {
      console.log('🔄 检测到回答数据更新，同步到列表:', updatedAnswer.id);
      syncAnswerData(updatedAnswer);
      
      // 清除 params，避免重复处理
      navigation.setParams({ updatedAnswer: undefined });
    }
  });

  return unsubscribe;
}, [navigation, answersCache, route?.params?.id, route?.params?.updatedAnswer]);
```

#### 1.2 实现 syncAnswerData 函数

```javascript
const syncAnswerData = (updatedAnswer) => {
  if (!updatedAnswer || !updatedAnswer.id) return;
  
  console.log('🔄 开始同步回答数据:', {
    answerId: updatedAnswer.id,
    isLiked: updatedAnswer.isLiked,
    isBookmarked: updatedAnswer.isBookmarked,
    isDisliked: updatedAnswer.isDisliked,
    likeCount: updatedAnswer.likeCount,
    bookmarkCount: updatedAnswer.bookmarkCount,
    dislikeCount: updatedAnswer.dislikeCount
  });
  
  // 更新所有排序方式的缓存中的该回答
  setAnswersCache(prevCache => {
    const newCache = { ...prevCache };
    
    Object.keys(newCache).forEach(sortBy => {
      const cacheData = newCache[sortBy];
      if (cacheData.list && cacheData.list.length > 0) {
        const answerIndex = cacheData.list.findIndex(a => a.id === updatedAnswer.id);
        
        if (answerIndex !== -1) {
          const updatedList = [...cacheData.list];
          updatedList[answerIndex] = {
            ...updatedList[answerIndex],
            // 用户状态
            isLiked: updatedAnswer.isLiked,
            liked: updatedAnswer.isLiked,
            isBookmarked: updatedAnswer.isBookmarked,
            bookmarked: updatedAnswer.isBookmarked,
            collected: updatedAnswer.isBookmarked,
            isDisliked: updatedAnswer.isDisliked,
            disliked: updatedAnswer.isDisliked,
            // 统计数据
            likeCount: updatedAnswer.likeCount,
            like_count: updatedAnswer.likeCount,
            likes: updatedAnswer.likeCount,
            bookmarkCount: updatedAnswer.bookmarkCount,
            bookmark_count: updatedAnswer.bookmarkCount,
            bookmarks: updatedAnswer.bookmarkCount,
            dislikeCount: updatedAnswer.dislikeCount,
            dislike_count: updatedAnswer.dislikeCount,
            dislikes: updatedAnswer.dislikeCount,
          };
          
          newCache[sortBy] = {
            ...cacheData,
            list: updatedList
          };
        }
      }
    });
    
    return newCache;
  });
  
  // 同步本地状态
  setAnswerLiked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isLiked || false
  }));
  
  setAnswerBookmarked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isBookmarked || false
  }));
  
  setAnswerDisliked(prev => ({
    ...prev,
    [updatedAnswer.id]: updatedAnswer.isDisliked || false
  }));
  
  console.log('✅ 回答数据同步完成');
};
```

### 2. 回答详情页 (AnswerDetailScreen.js)

#### 2.1 实现 syncDataToQuestionDetail 函数

```javascript
const syncDataToQuestionDetail = () => {
  if (!answerData || !navigation) return;
  
  console.log('🔄 准备同步数据到问题详情页:', {
    answerId: answerData.id,
    isLiked: answerLiked,
    isBookmarked: answerBookmarked,
    isDisliked: answerDisliked,
    likeCount: answerData.likeCount,
    bookmarkCount: answerData.bookmarkCount,
    dislikeCount: answerData.dislikeCount
  });
  
  // 获取导航栈中的问题详情页
  const routes = navigation.getState()?.routes;
  const questionDetailRoute = routes?.find(r => r.name === 'QuestionDetail');
  
  if (questionDetailRoute) {
    // 构建更新的回答数据
    const updatedAnswer = {
      id: answerData.id,
      // 用户状态
      isLiked: answerLiked,
      liked: answerLiked,
      isBookmarked: answerBookmarked,
      bookmarked: answerBookmarked,
      collected: answerBookmarked,
      isDisliked: answerDisliked,
      disliked: answerDisliked,
      // 统计数据
      likeCount: answerData.likeCount || 0,
      like_count: answerData.likeCount || 0,
      likes: answerData.likeCount || 0,
      bookmarkCount: answerData.bookmarkCount || 0,
      bookmark_count: answerData.bookmarkCount || 0,
      bookmarks: answerData.bookmarkCount || 0,
      dislikeCount: answerData.dislikeCount || 0,
      dislike_count: answerData.dislikeCount || 0,
      dislikes: answerData.dislikeCount || 0,
    };
    
    // 通过 navigation.navigate 传递更新的数据
    navigation.navigate('QuestionDetail', {
      ...questionDetailRoute.params,
      updatedAnswer: updatedAnswer
    });
    
    console.log('✅ 数据已同步到问题详情页');
  }
};
```

## 同步的数据字段

### 用户状态
- `isLiked` / `liked` - 是否已点赞
- `isBookmarked` / `bookmarked` / `collected` - 是否已收藏
- `isDisliked` / `disliked` - 是否已点踩

### 统计数据
- `likeCount` / `like_count` / `likes` - 点赞数
- `bookmarkCount` / `bookmark_count` / `bookmarks` - 收藏数
- `dislikeCount` / `dislike_count` / `dislikes` - 点踩数

> 注意：使用多个字段名是为了兼容不同的 API 响应格式

## 测试场景

### 场景 1：收藏操作同步
1. 进入问题详情页，查看某个回答（未收藏）
2. 点击该回答，进入回答详情页
3. 在回答详情页点击收藏按钮
4. 返回问题详情页
5. ✅ 验证：该回答显示为已收藏状态，收藏数 +1

### 场景 2：取消收藏操作同步
1. 进入问题详情页，查看某个回答（已收藏）
2. 点击该回答，进入回答详情页
3. 在回答详情页点击取消收藏按钮
4. 返回问题详情页
5. ✅ 验证：该回答显示为未收藏状态，收藏数 -1

### 场景 3：多次操作同步
1. 进入问题详情页
2. 点击回答 A，进入详情页，收藏
3. 返回，点击回答 B，进入详情页，收藏
4. 返回问题详情页
5. ✅ 验证：回答 A 和 B 都显示为已收藏状态

### 场景 4：缓存同步
1. 在问题详情页切换排序方式（精选/最新）
2. 点击某个回答，进入详情页，收藏
3. 返回问题详情页
4. 切换排序方式
5. ✅ 验证：在两种排序下，该回答都显示为已收藏状态

## 后续扩展

### 1. 点赞功能同步
在回答详情页实现点赞功能后，在 `handleAnswerLike` 成功回调中调用 `syncDataToQuestionDetail()`

### 2. 点踩功能同步
在回答详情页实现点踩功能后，在 `handleAnswerDislike` 成功回调中调用 `syncDataToQuestionDetail()`

### 3. 其他页面同步
如果有其他页面（如个人主页、收藏列表）也需要同步，可以：
- 使用相同的 params 传递机制
- 或者考虑引入全局状态管理（Context API / Redux）

## 注意事项

1. **避免重复处理**：在 focus listener 中处理完 `updatedAnswer` 后，立即清除该参数
2. **数据完整性**：确保传递的 `updatedAnswer` 包含所有必要的字段
3. **性能优化**：只更新变化的回答，不重新加载整个列表
4. **错误处理**：如果找不到对应的回答，记录日志但不影响用户体验
5. **多排序支持**：同时更新所有排序方式的缓存，确保切换排序时数据一致
