## 🎯 核心优化详解

### 批量加载机制

**优化前的问题**:
```
1. 加载问题详情 → 显示"补充(0)"、"回答(0)"
2. 加载补充列表 → 变为"补充(1)" ⚡闪烁
3. 加载回答列表 → 变为"回答(5)" ⚡闪烁
```

**优化后的流程**:
```javascript
// 1. 并行加载所有数据
const [detailResponse, supplementsResponse, answersResponse] = 
  await Promise.all([
    questionApi.getQuestionDetail(questionId),
    loadQuestionSupplements(questionId, 'featured', 1, true),
    answerApi.getAnswers(questionId, { sortBy: 'featured', pageNum: 1 })
  ]);

// 2. 批量更新状态（React自动批处理）
setQuestionData(detailResponse.data);
setSupplementsCache({ ...featured: { loaded: true, ... } });
setAnswersCache({ ...featured: { loaded: true, ... } });

// 3. 启动淡入动画
setTimeout(() => {
  Animated.timing(fadeAnim, { toValue: 1, duration: 300 }).start();
}, 0);
```

### 条件显示数量

**关键代码**:
```javascript
const answerTabs = React.useMemo(() => {
  // 检查初始数据是否已加载
  const initialDataLoaded = 
    supplementsCache.featured.loaded &&

}, [t, supplementsList.length, answersTotal, 
    supplementsCache.featured.loaded, 
    answersCache.featured.loaded]);
```

### 防抖机制

**回答收藏操作**:
```javascript
const handleAnswerCollect = async (answerId) => {
  // 1. 防止重复请求
  if (answerCollectLoading[answerId]) return;
  
  // 2. 清除之前的防抖定时器
  clearTimeout(answerCollectDebounce[answerId]);
  
  // 3. 立即更新UI（乐观更新）
  setAnswerBookmarked(prev => ({ ...prev, [answerId]: !prev[answerId] }));
  
  // 4. 300ms后执行实际请求
  const timer = setTimeout(async () => {
    const response = newState 
      ? await answerApi.collectAnswer(answerId)
      : await answerApi.uncollectAnswer(answerId);
    
    if (response.code !== 200) {
      // 失败时回滚UI
      setAnswerBookmarked(prev => ({ ...prev, [answerId]: currentState }));
    }
  }, 300);
};
```
