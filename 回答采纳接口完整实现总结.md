# 回答采纳接口完整实现总结

## 接口规范

- **接口地址**: `/app/content/answer/question/{questionId}/accept/{answerId}`
- **请求方式**: `PUT`
- **请求参数**: 
  - `questionId`: 当前问题的ID（int类型）
  - `answerId`: 当前回答的ID（int类型）
- **返回数据**: `{"code": 200, "msg": "", "data": {}}`

### 4. 采纳处理函数

```javascript
// 处理回答采纳/取消采纳
const handleAnswerAdopt = async (answerId) => {
  // 参数验证
  if (!answerId) {
    showToast('操作失败：回答ID不存在', 'error');
    return;
  }

  const questionId = route?.params?.id;
  if (!questionId) {
    showToast('操作失败：问题ID不存在', 'error');
    return;
  }

  // 防止重复请求
  if (answerAdoptLoading[answerId]) {
    return;
  }

  // 获取当前采纳状态
  const currentState = answerAdopted[answerId] !== undefined 
    ? answerAdopted[answerId] 
    : (answersList.find(a => a.id === answerId)?.adopted || false);

  // 如果已经采纳，不允许取消采纳（根据需求）
  if (currentState) {
    showToast('该回答已被采纳，无法更改状态', 'warning');
    return;
  }

  try {
    // 设置加载状态
    setAnswerAdoptLoading(prev => ({ ...prev, [answerId]: true }));

    // 调用API
    const response = await answerApi.adoptAnswer(questionId, answerId);
    
    if (response && response.code === 200) {
      // 更新UI状态
      setAnswerAdopted(prev => ({ ...prev, [answerId]: true }));
      showToast('已采纳该回答', 'success');
      
      // 刷新回答列表
      setAnswersCache(prevCache => ({
        ...prevCache,
        [answersSortBy]: { ...prevCache[answersSortBy], loaded: false }
      }));
      loadAnswersList(questionId, true, answersSortBy);
    } else {
      showToast(response?.msg || '采纳失败', 'error');
    }
  } catch (error) {
    showToast('网络错误，请稍后重试', 'error');
  } finally {
    // 清除加载状态
    setAnswerAdoptLoading(prev => ({ ...prev, [answerId]: false }));
  }
};
```

## 🚨 问题排查和修复

### 遇到的问题
**错误信息**: `{"code": 500, "msg": "问题与回答不匹配"}`

### 问题分析
1. **参数类型问题**: 后端可能期望数字类型参数
2. **数据关联问题**: 回答ID可能不属于指定的问题ID
3. **接口调用方式问题**: 可能需要POST而不是PUT请求

### 已实施的修复

#### 1. 参数类型转换和验证
```javascript
// 确保参数为数字类型
const numQuestionId = parseInt(questionId, 10);
const numAnswerId = parseInt(answerId, 10);

// 参数验证
if (!Number.isInteger(numQuestionId) || numQuestionId <= 0) {
  throw new Error('无效的问题ID');
}
if (!Number.isInteger(numAnswerId) || numAnswerId <= 0) {
  throw new Error('无效的回答ID');
}
```

#### 2. 请求方式兼容
```javascript
try {
  // 尝试PUT请求
  return await contentApiClient.put(url);
} catch (error) {
  // 如果PUT失败，尝试POST请求
  return await contentApiClient.post(url);
}
```

## 使用说明

1. 用户在问题详情页面的回答列表中看到采纳按钮
2. 点击采纳按钮后，按钮显示加载状态
3. 请求成功后，按钮变为"已采纳"状态且不可再点击
4. 回答列表会自动刷新显示最新状态
5. 如果请求失败，会显示错误提示
