# 服务器切换功能说明

## 功能概述

为了方便开发阶段在两个服务器地址之间快速切换，我们在"我的"页面添加了一个服务器切换组件。

## 可切换的服务器

1. **开发服务器 (Server1)**: `http://123.144.149.59:30560`
2. **生产服务器 (Server2)**: `http://8.146.230.62:8080`

## 使用方法

1. 打开应用，进入"我的"页面（底部导航栏最右侧）
2. 向下滚动，在功能菜单下方可以看到"开发工具 - 服务器切换"卡片
3. 卡片中显示两个服务器选项，当前使用的服务器会有绿色的"当前"标签和对勾图标
4. 点击另一个服务器选项，会弹出确认对话框
5. 确认后，应用会自动重新加载，所有API请求将使用新的服务器地址

## 注意事项

- ⚠️ 此功能仅在开发模式下可见（`__DEV__ === true`）
- ⚠️ 切换服务器后应用会自动重新加载，未保存的数据会丢失
- ⚠️ 切换后的服务器选择会被保存，下次启动应用时会继续使用
- ⚠️ 生产环境构建的应用中不会显示此功能

## 技术实现

### 相关文件

1. **`src/components/ServerSwitcher.js`** - 服务器切换UI组件
2. **`src/utils/serverSwitcher.js`** - 服务器切换逻辑
3. **`src/config/env.js`** - 环境配置和动态服务器选择
4. **`src/screens/ProfileScreen.js`** - 集成了ServerSwitcher组件

### 工作原理

1. 服务器选择保存在 AsyncStorage 中（key: `@app_server_selection`）
2. 应用启动时从 AsyncStorage 读取服务器选择
3. 所有API请求通过 `env.js` 中的 `API_SERVER_CONFIG` 动态获取服务器地址
4. 切换服务器时，保存新选择并调用 `Updates.reloadAsync()` 重新加载应用

### 如何移除此功能

如果不再需要此功能，可以：

1. 从 `ProfileScreen.js` 中删除 `<ServerSwitcher />` 组件
2. 删除 `src/components/ServerSwitcher.js` 文件
3. （可选）删除 `src/utils/serverSwitcher.js` 文件
