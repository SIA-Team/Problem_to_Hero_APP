# 模拟生产环境配置说明

## 🎯 快速答案

**问题**：构建生产版本前需要关闭模拟生产环境配置吗？

**答案**：❌ 不需要！可以直接构建！

---

## 🔍 原理解释

### 配置文件

**文件位置**：`src/config/debugMode.js`

```javascript
export const SIMULATE_PRODUCTION = true;  // 当前设置
```

### 环境判断逻辑

**文件位置**：`src/config/env.js`

```javascript
const getEnvVars = () => {
  // 1️⃣ 如果开启了模拟生产环境，返回生产配置
  if (SIMULATE_PRODUCTION) {
    console.log('🎭 使用生产环境配置（模拟模式）');
    return ENV.prod;
  }
  
  // 2️⃣ 通过 __DEV__ 判断是否为开发环境
  if (__DEV__) {
    return ENV.dev;
  }
  
  // 3️⃣ 默认返回生产环境（生产构建时）
  return ENV.prod;
};
```

### 关键点

**`__DEV__` 变量**：
- React Native 自动设置
- 开发环境：`__DEV__ = true`
- 生产构建：`__DEV__ = false`

**执行顺序**：
1. 先检查 `SIMULATE_PRODUCTION`
2. 再检查 `__DEV__`
3. 最后返回默认值

**但是**：生产构建时，`__DEV__ = false`，代码会跳过前两个检查，直接返回 `ENV.prod`！

## 📊 不同场景的行为

### 场景 1: 开发服务器（模拟关闭）

```javascript
__DEV__ = true
SIMULATE_PRODUCTION = false
```

**执行流程**：
1. ❌ `SIMULATE_PRODUCTION` 为 false，跳过
2. ✅ `__DEV__` 为 true，返回 `ENV.dev`

**结果**：使用开发环境配置

---

### 场景 2: 开发服务器（模拟开启）⭐ 当前状态

```javascript
__DEV__ = true
SIMULATE_PRODUCTION = true
```

**执行流程**：
1. ✅ `SIMULATE_PRODUCTION` 为 true，返回 `ENV.prod`

**结果**：使用生产环境配置（模拟）

---

### 场景 3: 生产构建（模拟关闭）

```javascript
__DEV__ = false
SIMULATE_PRODUCTION = false
```

**执行流程**：
1. ❌ `SIMULATE_PRODUCTION` 为 false，跳过
2. ❌ `__DEV__` 为 false，跳过
3. ✅ 返回默认值 `ENV.prod`

**结果**：使用生产环境配置

---

### 场景 4: 生产构建（模拟开启）⭐ 你担心的场景

```javascript
__DEV__ = false
SIMULATE_PRODUCTION = true
```

**执行流程**：
1. ✅ `SIMULATE_PRODUCTION` 为 true，返回 `ENV.prod`

**结果**：使用生产环境配置

## 🎯 实际影响

### 开发环境（npm run start）

| SIMULATE_PRODUCTION | 使用的配置 | 说明 |
|-------------------|----------|------|
| false | ENV.dev | 开发配置 |
| true | ENV.prod | 生产配置（模拟） |

**用途**：在开发时测试生产环境的行为

---

### 生产构建（eas build）

| SIMULATE_PRODUCTION | 使用的配置 | 说明 |
|-------------------|----------|------|
| false | ENV.prod | 生产配置 |
| true | ENV.prod | 生产配置 |

**结论**：`SIMULATE_PRODUCTION` 的值不影响生产构建！

## 💡 为什么设计成这样？

### 设计目的

1. **开发时模拟生产环境**
   - 测试生产环境的 API
   - 测试生产环境的行为
   - 避免频繁构建生产版本

2. **生产构建时自动使用生产配置**
   - 不需要手动修改配置
   - 避免人为错误
   - 简化构建流程

### 安全机制

**即使忘记关闭 `SIMULATE_PRODUCTION`**：
- ✅ 生产构建仍然正确
- ✅ 不会影响生产环境
- ✅ 不需要担心配置错误

## ⚠️ 注意事项

### 唯一需要注意的

**如果你修改了 `ENV.dev` 和 `ENV.prod` 的配置**：

```javascript
const ENV = {
  dev: {
    apiUrl: 'http://dev-server',  // 开发服务器
  },
  prod: {
    apiUrl: 'http://prod-server',  // 生产服务器
  }
};
```

**确保**：
- `ENV.prod` 指向正确的生产服务器
- `ENV.dev` 指向正确的开发服务器

**当前配置**：
```javascript
const ENV = {
```
