# 智能缓存失效机制说明

## 问题背景
用户担心缓存机制会导致看不到后端数据的最新更新，需要一个能自动检测和更新数据的智能缓存系统。

## 解决方案：多层缓存失效机制

### 1. 时间过期机制
```javascript
const CACHE_EXPIRE_TIME = 5 * 60 * 1000; // 5分钟缓存过期时间

const isCacheExpired = (cacheData) => {
  if (!cacheData.lastUpdated) return true;
  return Date.now() - cacheData.lastUpdated > CACHE_EXPIRE_TIME;
};
```

**工作原理**：
- 每个缓存记录创建时间戳
- 超过5分钟自动标记为过期
- 过期后下次访问会重新请求

### 2. 定时静默刷新
```javascript
const AUTO_REFRESH_INTERVAL = 30 * 1000; // 30秒检查间隔

useEffect(() => {
  const interval = setInterval(() => {
    const currentCache = answersCache[answersSortBy];
    if (currentCache.loaded && isCacheExpired(currentCache)) {
      silentRefreshCache(questionId, answersSortBy);
    }
  }, AUTO_REFRESH_INTERVAL);
  
  return () => clearInterval(interval);
}, [answersSortBy, answersCache]);
```

**工作原理**：
- 每30秒检查当前显示的缓存是否过期
- 过期则后台静默刷新，不影响用户操作
- 检测到数据变化时自动更新并提示用户

### 3. 页面焦点检查
```javascript
useEffect(() => {
  const unsubscribe = navigation.addListener('focus', () => {
    Object.keys(answersCache).forEach(sortBy => {
      const cacheData = answersCache[sortBy];
      if (cacheData.loaded && isCacheExpired(cacheData)) {
        silentRefreshCache(questionId, sortBy);
      }
    });
  });
  
  return unsubscribe;
}, [navigation, answersCache]);
```

**工作原理**：
- 用户切换回页面时检查所有缓存
- 过期的缓存自动后台更新
- 确保用户看到的是相对新鲜的数据

### 4. 智能数据变化检测
```javascript
const silentRefreshCache = async (questionId, sortBy) => {
  const response = await answerApi.getAnswers(questionId, { sortBy, pageNum: 1, pageSize: 10 });
```

- 避免无意义的UI刷新

### 5. 操作触发缓存清除
```javascript
// 发布回答成功后
if (response && response.code === 200) {

```

**工作原理**：
- 用户操作（发布回答、点赞等）后立即清除缓存
- 确保用户能立即看到自己操作的结果
- 避免缓存导致的数据不一致

## 缓存更新策略对比

| 触发方式 | 更新时机 | 用户感知 | 数据新鲜度 | 性能影响 |
|---------|---------|---------|-----------|----------|
| **手动刷新** | 用户下拉 | 有loading | 最新 | 低 |
| **时间过期** | 5分钟后访问 | 有loading | 较新 | 低 |
| **定时检查** | 30秒间隔 | 无感知 | 新 | 极低 |
| **焦点检查** | 页面切换回来 | 无感知 | 新 | 极低 |
| **操作触发** | 立即 | 有loading | 最新 | 低 |

## 用户体验优化

### 1. 静默更新
- 后台检查不影响当前操作
- 只在检测到变化时才更新UI
- 提供轻量级的更新提示

### 2. 智能提示
```javascript
if (hasChanges && sortBy === answersSortBy) {
  showToast('回答列表已更新', 'info');
}
```
- 只在当前显示的排序有更新时提示
- 使用info级别，不打断用户操作

### 3. 渐进式更新
- 保持用户当前的滚动位置
- 新数据平滑插入，不突兀跳转
- 维持用户的操作连续性

## 网络优化

### 1. 请求去重
- 同一时间只允许一个刷新请求
- 避免重复的网络请求

### 2. 错误处理
- 静默刷新失败不影响用户体验
- 保留旧缓存数据作为降级方案

### 3. 数据对比优化
- 只对比关键字段（总数、前几条数据）
- 避免深度对比带来的性能开销
