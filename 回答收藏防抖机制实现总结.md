# 回答收藏防抖机制实现总结

## 问题背景
用户担心频繁点击收藏按钮会给服务器造成压力，需要实现一个既能提供良好用户体验又能保护服务器的收藏功能。

## 解决方案：多重保护机制

### 1. 防抖机制 (Debounce)
```javascript
const debounceTimer = setTimeout(async () => {
  // 实际的API请求
}, 300); // 300ms防抖延迟
```

**工作原理**：
- 用户点击后等待300ms再发送请求
- 如果300ms内再次点击，取消之前的请求，重新计时
- 只有最后一次点击会真正发送请求

### 2. 请求去重机制
```javascript
if (answerCollectLoading[answerId]) {
  console.log('🚫 收藏请求进行中，忽略重复点击:', answerId);
  return;
}
```

**工作原理**：
- 每个回答维护独立的加载状态
- 请求进行中时忽略新的点击
- 请求完成后才允许下次操作

### 3. 乐观更新 (Optimistic Update)
```javascript
// 立即更新UI状态，提供即时反馈
const currentState = answerBookmarked[answerId] || false;
const newState = !currentState;

setAnswerBookmarked(prev => ({
  ...prev,
  [answerId]: newState
}));
```

**工作原理**：
- 点击后立即更新UI，用户看到即时反馈
- 后台发送请求验证操作
- 如果请求失败，回滚UI状态

### 4. 错误回滚机制
```javascript
if (response && response.code === 200) {
  // 成功：保持UI状态
  showToast(newState ? '已收藏' : '已取消收藏', 'success');
} else {
  // 失败：回滚UI状态
  setAnswerBookmarked(prev => ({
    ...prev,
    [answerId]: currentState // 回滚到原状态
  }));
  showToast(response?.msg || '操作失败', 'error');
}
```

**工作原理**：
- 请求成功：保持乐观更新的状态
- 请求失败：回滚到操作前的状态
- 网络错误：同样回滚状态

## 完整的用户交互流程

### 正常流程
1. **用户点击收藏按钮**
2. **UI立即响应**：图标变色，数字+1
3. **显示加载状态**：按钮略微透明，显示loading图标
4. **等待300ms**：防抖延迟
5. **发送API请求**
6. **请求成功**：显示"已收藏"提示，保持UI状态
7. **清除加载状态**：按钮恢复正常

### 频繁点击流程
1. **用户快速点击多次**
2. **UI跟随最后一次点击状态**
3. **前面的请求被取消**
4. **只有最后一次点击生效**
5. **发送一次API请求**

### 网络错误流程
1. **用户点击收藏按钮**
2. **UI立即响应**
3. **发送请求失败**
4. **UI回滚到原状态**
5. **显示错误提示**

## 性能优化特性

### 1. 请求合并
- 300ms内的多次点击只发送一次请求
- 大大减少服务器压力

### 2. 状态管理优化
```javascript
const [answerCollectLoading, setAnswerCollectLoading] = useState({}); // 按回答ID分别管理
const [answerCollectDebounce, setAnswerCollectDebounce] = useState({}); // 防抖定时器管理
```

- 每个回答独立管理状态
- 不同回答的操作互不干扰

### 3. 内存清理
```javascript
useEffect(() => {
  return () => {
    Object.values(answerCollectDebounce).forEach(timer => {
      if (timer) clearTimeout(timer);
    });
  };
}, [answerCollectDebounce]);
```

- 组件卸载时清理所有定时器
- 避免内存泄漏

## API接口设计

### 接口地址
```
POST /app/content/answer/{id}/collect
```

### 请求参数
- `id`: 回答ID（路径参数）

### 返回格式
```json
{
  "code": 200,
  "msg": "",
  "data": {
    "collected": true  // 可选：返回实际收藏状态
  }
}
```

### 接口特点
- **幂等性**：同一个接口处理收藏和取消收藏
- **状态切换**：每次调用切换收藏状态
- **简单易用**：不需要额外参数

## 服务器保护机制

### 1. 请求频率控制
```
正常情况：1次点击 = 1次请求
防抖优化：10次快速点击 = 1次请求
减少请求：90%
```

### 2. 并发控制
- 同一回答同时只能有一个收藏请求
- 避免并发请求导致的状态混乱

### 3. 超时处理
- 请求超时自动回滚状态
- 避免长时间占用连接
