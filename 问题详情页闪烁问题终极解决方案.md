# 问题详情页闪烁问题终极解决方案

## 问题分析

### 闪烁的根本原因

1. **分步渲染导致的视觉跳动**
   ```
   时间轴：
   0ms:   显示骨架屏
   200ms: 问题详情加载完成 → 渲染页面（补充 (0)，回答 (0)）← 第一次渲染
   400ms: 补充列表加载完成 → 更新标签（补充 (1)）← 第二次渲染（闪烁！）
   600ms: 回答列表加载完成 → 更新标签（回答 (5)）← 第三次渲染（闪烁！）
   ```

2. **多次状态更新**
   - 问题详情 → 触发渲染
   - 补充列表 → 触发渲染
   - 回答列表 → 触发渲染
   - 每次渲染都会导致标签文本变化

3. **异步数据加载的时间差**
   - 不同接口返回时间不同
   - 每个接口返回都会更新状态
   - 状态更新导致重新渲染

## 终极解决方案

### 核心思路：批量加载 + 一次性渲染

**关键点**：
1. ✅ 并行加载所有数据
2. ✅ 等待所有数据都返回
3. ✅ 批量更新所有状态
4. ✅ 一次性渲染完整内容

### 实现代码

```javascript
useEffect(() => {
  const fetchQuestionDetail = async () => {
    try {
      setLoading(true);
      
      // 步骤1: 并行加载所有数据
      const [detailResponse, supplementsResponse, answersResponse] = await Promise.all([
        questionApi.getQuestionDetail(questionId),
        loadQuestionSupplements(questionId, 'featured', 1, true),
        answerApi.getAnswers(questionId, { sortBy: 'featured', pageNum: 1, pageSize: 10 })
      ]);
      
      // 步骤2: 批量更新所有状态（React 自动批处理）
      setQuestionData(detailResponse.data);
      setSupplementsCache(/* 更新补充缓存 */);
      setAnswersCache(/* 更新回答缓存 */);
      
      // 步骤3: 启动淡入动画
      Animated.timing(fadeAnim, {
        toValue: 1,
        duration: 300,
        useNativeDriver: true,
      }).start();
      
    } finally {
      setLoading(false);
    }
  };
  
  fetchQuestionDetail();
}, [route?.params?.id]);
```

## 技术细节

### 1. Promise.all 并行加载

```javascript
const [detailResponse, supplementsResponse, answersResponse] = await Promise.all([
  questionApi.getQuestionDetail(questionId),
  loadQuestionSupplements(questionId, 'featured', 1, true),
  answerApi.getAnswers(questionId, { sortBy: 'featured', pageNum: 1, pageSize: 10 })
]);
```

**优点**：
- 并行请求，不浪费时间
- 等待所有请求完成
- 总时间 = max(请求1, 请求2, 请求3)

### 2. React 自动批处理

React 18 会自动批处理多个状态更新：

```javascript
// 这些更新会被批处理为一次渲染
setQuestionData(data1);
setSupplementsCache(data2);
setAnswersCache(data3);
```

### 3. requestAnimationFrame 优化

```javascript
requestAnimationFrame(() => {
  Animated.timing(fadeAnim, {
    toValue: 1,
    duration: 300,
    useNativeDriver: true,
  }).start();
});
```

**作用**：
- 确保动画在下一帧开始
- 避免与渲染冲突
- 更流畅的视觉效果

### 4. 直接更新缓存状态

不再调用 `loadSupplementsList` 和 `loadAnswersList`，而是直接更新缓存：

```javascript
setSupplementsCache(prevCache => ({
  ...prevCache,
  featured: {
    list: supplements,
    total: supplements.length,
    pageNum: supplements.length < 10 ? 1 : 2,
    hasMore: supplements.length >= 10,
    loaded: true,
    lastUpdated: Date.now()
  }
}));
```

**优点**：
- 避免中间状态
- 一次性设置完整数据
- 不触发额外的渲染
