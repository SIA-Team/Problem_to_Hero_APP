## 📡 接口规格

### 请求
```
GET /app/content/question/drafts
参数：
- pageNum: 1 (页码)
- pageSize: 10 (每页数量)
```

### 响应
```json
{
  "code": 0,
  "msg": "获取成功", 
  "data": {
    "total": 3,
    "rows": [
      {
        "id": 1,
        "type": 0,
        "categoryId": 201,
        "title": "草稿标题",
        "description": "草稿描述",
        "createTime": "2024-03-10T10:30:00Z",
        "updateTime": "2024-03-10T10:30:00Z"
      }
    ]
  }
}
```

## 🎨 UI 功能

### 1. 数据显示
- **问题类型标签**：根据 type 字段显示（0=公开问题，1=悬赏问题，2=定向问题）
- **标题**：显示草稿标题，无标题时显示"无标题"
- **时间**：格式化显示创建时间

### 2. 交互功能
- **点击草稿**：跳转到发布页面，传递草稿数据用于编辑
- **删除草稿**：保留原有的删除确认逻辑

### 3. 状态处理
- **加载状态**：显示加载指示器和"加载中..."文本
- **空状态**：显示空状态图标和提示文本
- **错误处理**：在控制台输出错误信息

## 🔄 数据流程

1. **用户点击"我的草稿"** → 打开弹窗
2. **调用 loadDraftsList()** → 请求接口数据
3. **更新状态** → 渲染草稿列表
4. **用户点击草稿** → 跳转到编辑页面
