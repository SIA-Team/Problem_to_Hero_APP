# 定向问题悬赏设置更新说明

## 更新内容

将发布页面中定向问题的悬赏金额设置模块与公开问题保持一致。

## 具体修改

### 1. 标题统一
- **修改前**: "设置奖赏金额 *"（带必填标记）
- **修改后**: "设置悬赏金额"（无必填标记）

### 2. 描述文案统一
- **修改前**: "可以设置为 $0（不设悬赏）或任意金额"
- **修改后**: "可以设置为 $0（不设悬赏）或任意金额"（保持一致）

### 3. 输入框提示统一
- **修改前**: "输入金额（最低$0）"
- **修改后**: "输入金额（可以是$0）"

### 4. 验证逻辑优化
- **修改前**: 验证金额不能小于$0，如果小于$0则提示错误
- **修改后**: 移除验证逻辑，允许任意金额（包括$0和空值）

## 功能说明

### 公开问题悬赏设置
```
标题: 设置悬赏金额
描述: 可以设置为 $0（不设悬赏）或任意金额
快捷金额: $10, $20, $50, $100
自定义输入: 输入金额（可以是$0）
验证: 无强制要求，可以为空或$0
```

### 定向问题悬赏设置（更新后）
```
标题: 设置悬赏金额
描述: 可以设置为 $0（不设悬赏）或任意金额
快捷金额: $10, $20, $50, $100
自定义输入: 输入金额（可以是$0）
验证: 无强制要求，可以为空或$0
```

### 悬赏问题悬赏设置（保持不变）
```
标题: 设置悬赏金额 *
描述: 悬赏金额不能小于 $1
快捷金额: $10, $20, $50, $100
自定义输入: 输入金额（最低$1）
验证: 必填，最低$1
```

## 用户体验改进

1. **一致性**: 公开问题和定向问题的悬赏设置UI和交互完全一致
2. **灵活性**: 定向问题可以不设置悬赏（$0或留空），更加灵活
3. **清晰性**: 去掉必填标记，用户不会误以为定向问题必须设置悬赏

## 技术细节

### 修改的文件
- `src/screens/PublishScreen.js`

### 修改的代码位置
1. 第1204-1222行: 定向问题悬赏金额UI
2. 第613-621行: 定向问题验证逻辑

### 数据处理
- 如果用户不输入悬赏金额，`targetedReward` 为空字符串
- 在发布时，如果 `targetedReward` 为空或无效，`bountyAmount` 会使用默认值5000（50元）
- 如果用户输入了有效金额，会转换为分（乘以100）后发送给后端

## 注意事项

1. 此修改只影响定向问题的悬赏设置UI和验证逻辑
2. 不影响悬赏问题的设置（悬赏问题仍然要求最低$1）
3. 后端接口保持不变，只是前端验证逻辑调整
4. 可以通过热更新发布，无需重新构建应用
