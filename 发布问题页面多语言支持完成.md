# 发布问题页面多语言支持完成

## 修改内容

已成功为发布问题页面（PublishScreen.js）添加完整的多语言支持，现在页面会根据系统语言自动显示中文或英文。

## 修改的文件

### 1. 翻译文件

#### `src/i18n/locales/zh.json`
添加了完整的中文翻译，包括：
- 页面标题和按钮
- 问题类型（公开问题、悬赏问题、定向问题）
- 表单标签和占位符
- 提示信息和错误消息
- 可见性选项
- 答案设置选项

#### `src/i18n/locales/en.json`
添加了完整的英文翻译，与中文翻译一一对应。

### 2. 页面组件

#### `src/screens/PublishScreen.js`
- 导入了 `useTranslation` hook
- 使用 `t()` 函数替换所有硬编码的中文文本
- 修改了 `questionTypes` 数组，使用翻译键而不是硬编码文本

## 翻译键结构

```
publish.
├── title                    # 发布问题
├── saveDraft               # 存草稿
├── selectQuestionType      # 选择问题类型
├── questionTypes.
│   ├── public              # 公开问题
│   ├── publicDesc          # 公开提问
│   ├── reward              # 悬赏问题
│   ├── rewardDesc          # 付费求答
│   ├── targeted            # 定向问题
│   └── targetedDesc        # 指定回答
├── category                # 问题类别
├── setRewardAmount         # 设置悬赏金额
├── inviteExperts           # 邀请回答专家
├── questionTitle           # 问题标题
├── questionDescription     # 问题描述
├── addImages               # 添加图片
├── selectTopics            # 选择话题
├── answerSettings          # 答案设置
├── visibility.
│   ├── everyone            # 所有人
│   ├── everyoneDesc        # 所有用户都可以看到这个问题
│   ├── followers           # 仅关注我的人
│   ├── followersDesc       # 只有关注你的用户可以看到
│   ├── onlyMe              # 仅自己
│   └── onlyMeDesc          # 只有你自己可以看到这个问题
└── toasts.
    ├── maxTopics           # 最多选择3个话题
    ├── enterTitle          # 请输入问题标题
    ├── publishSuccess      # 问题发布成功
    └── ...                 # 其他提示消息
```

## 测试建议

1. 切换系统语言到英文，检查页面是否显示英文
2. 切换系统语言到中文，检查页面是否显示中文
3. 测试所有功能：
   - 选择问题类型
   - 选择分类
   - 输入标题和描述
   - 添加图片
   - 选择话题
   - 设置答案可见性
   - 发布问题
4. 检查所有提示消息是否正确显示对应语言

## 注意事项

- 所有用户可见的文本都已使用多语言系统
- 控制台日志和调试信息保持中文（不影响用户体验）
- 分类颜色映射中的中文键保持不变（这是数据映射，不是UI文本）
- 默认位置值保持中文（这是数据值，不是UI显示）
