# 回答列表字段映射修复总结

## 问题描述
回答列表中的点赞数、评论数等显示为 "NaN"，原因是接口返回的字段名与代码中使用的字段名不匹配。

## 问题分析
- **原代码使用的字段**：`likes`, `comments`, `shares`, `bookmarks`, `dislikes`, `author`, `avatar`
- **接口可能返回的字段**：`likeCount`, `commentCount`, `shareCount`, `bookmarkCount`, `dislikeCount`, `userName`, `userAvatar`

## 修复方案
使用安全的字段访问模式，支持多种可能的字段名：

### 1. 点赞数修复
```javascript
// 修复前
{answer.likes + (answerLiked[answer.id] ? 1 : 0)}

// 修复后
{(answer.likeCount || answer.like_count || answer.likes || 0) + (answerLiked[answer.id] ? 1 : 0)}
```

### 2. 评论数修复
```javascript
// 修复前
{answer.comments}

// 修复后
{answer.commentCount || answer.comment_count || answer.comments || 0}
```

### 3. 分享数修复
```javascript
// 修复前
{answer.shares || 34}

// 修复后
{answer.shareCount || answer.share_count || answer.shares || 34}
```

### 4. 收藏数修复
```javascript
// 修复前
{(answer.bookmarks || 89) + (answerBookmarked[answer.id] ? 1 : 0)}

// 修复后
{(answer.bookmarkCount || answer.bookmark_count || answer.bookmarks || 89) + (answerBookmarked[answer.id] ? 1 : 0)}
```

### 5. 踩数修复
```javascript
// 修复前
{answer.dislikes || 3}

// 修复后
{(answer.dislikeCount || answer.dislike_count || answer.dislikes || 3) + (answerDisliked[answer.id] ? 1 : 0)}
```

### 6. 作者信息修复
```javascript
// 修复前
<Avatar uri={answer.avatar} name={answer.author} size={24} />
<Text style={styles.answerAuthor}>{answer.author}</Text>

// 修复后
<Avatar uri={answer.userAvatar || answer.avatar} name={answer.userName || answer.userNickname || answer.author || '匿名用户'} size={24} />
<Text style={styles.answerAuthor}>{answer.userName || answer.userNickname || answer.author || '匿名用户'}</Text>
```

## 字段映射优先级

### 数值字段
1. **驼峰命名**：`likeCount`, `commentCount`, `shareCount`, `bookmarkCount`, `dislikeCount`
2. **下划线命名**：`like_count`, `comment_count`, `share_count`, `bookmark_count`, `dislike_count`
3. **原字段名**：`likes`, `comments`, `shares`, `bookmarks`, `dislikes`
4. **默认值**：0 或合理的默认数字

### 用户信息字段
1. **用户名**：`userName` → `userNickname` → `author` → `'匿名用户'`
2. **头像**：`userAvatar` → `avatar`

## 优势
1. **兼容性强**：支持多种API字段命名规范
2. **容错性好**：字段不存在时使用默认值，避免NaN
3. **可维护性**：统一的字段访问模式
4. **向后兼容**：保持对原字段名的支持
